# QMS子系统文档总览

## 1. 子系统概述
QMS（Quality Management System）质量管理系统是GMP系统的核心组件，负责全面管理企业的质量活动，包括偏差管理、变更控制、CAPA管理、投诉管理等，确保产品质量符合GMP规范和法规要求。

## 3. 核心功能模块

- **偏差管理**：偏差识别、记录、调查、评估和关闭
- **变更控制**：变更申请、风险评估、审批、实施和验证
- **CAPA管理**：纠正预防措施制定、实施、验证和效果评估
- **投诉管理**：客户投诉接收、调查、处理和反馈
- **质量事件管理**：质量事件分类、跟踪和趋势分析
- **供应商质量管理**：供应商评估、审计和资质管理
- **内部审计管理**：审计计划、执行、报告和跟踪
- **质量体系文档管理**：SOP、质量手册等文档的版本控制和访问管理

## 4. 技术架构

QMS子系统采用微服务架构设计，主要技术栈包括：

- **后端技术**：Java、Spring Boot、Spring Cloud
- **数据库**：PostgreSQL（主数据库）、MongoDB（非结构化数据）
- **前端技术**：React、Ant Design
- **API设计**：RESTful API
- **认证授权**：集成AUTH-SYS子系统的JWT认证机制
- **工作流引擎**：Camunda BPM（支持复杂审批流程）
- **文档管理**：集成EDMS子系统的文档存储和检索功能

## 5. 关键业务流程

- **偏差处理流程**：偏差发现 → 初步评估 → 详细调查 → 根本原因分析 → 纠正措施制定 → 实施 → 验证 → 关闭
- **变更控制流程**：变更申请 → 变更评估 → 风险分析 → 审批 → 实施计划制定 → 执行 → 验证 → 关闭
- **CAPA流程**：问题识别 → 调查 → 根本原因分析 → 措施制定 → 实施 → 验证 → 效果评估 → 关闭

## 6. 系统接口

QMS子系统与其他子系统的主要接口包括：

- **与AUTH-SYS集成**：用户认证、授权和角色管理
- **与MES集成**：生产数据关联和质量问题追溯
- **与LIMS集成**：测试数据关联和质量指标分析
- **与EDMS集成**：SOP和质量文档管理
- **与CRM-SALES集成**：客户投诉信息共享
- **与EQUIPMENT-MANAGEMENT集成**：设备相关质量问题管理
