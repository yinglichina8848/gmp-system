# 质量管理子系统用户指南

## 1. 文档概述

本文档是质量管理子系统(QMS)的用户操作指南，旨在帮助用户了解系统功能，掌握基本操作流程，有效地使用系统进行质量管理工作。本指南适用于系统的各类用户，包括系统管理员、质量经理、QA专员、部门主管和普通员工等。

## 2. 系统简介

### 2.1 系统概述

质量管理子系统是GMP系统的核心组成部分，用于实现制药企业质量管理的全过程管理。系统提供偏差管理、CAPA管理、变更控制、审计管理和投诉管理等核心功能模块，支持质量管理的电子化、规范化和流程化。

### 2.2 系统登录

1. **访问地址**：在浏览器中输入系统的URL地址
2. **登录界面**：输入用户名和密码，点击"登录"按钮
3. **身份验证**：系统进行身份验证，验证通过后进入系统主界面
4. **忘记密码**：如忘记密码，点击"忘记密码"链接，按照提示进行密码重置

### 2.3 主界面介绍

系统主界面主要由以下部分组成：

1. **顶部导航栏**：显示系统名称、用户信息、消息通知和退出按钮
2. **左侧菜单**：显示系统的功能模块导航菜单
3. **主内容区**：显示当前功能模块的具体内容和操作界面
4. **底部状态栏**：显示系统状态信息和版权信息

## 3. 偏差管理

### 3.1 偏差记录创建

1. **进入偏差管理模块**：在左侧菜单中点击"偏差管理"→"偏差记录"
2. **创建新偏差**：点击页面右上角的"创建偏差"按钮
3. **填写偏差信息**：
   - 标题：输入偏差的简要描述
   - 描述：详细描述偏差的情况
   - 类型：选择偏差类型（系统偏差、过程偏差、产品偏差、设备偏差等）
   - 严重程度：选择偏差的严重程度（轻微、中等、严重）
   - 发生日期：选择偏差发生的日期和时间
   - 发生地点：输入偏差发生的地点
   - 受影响产品/批次：输入受影响的产品或批次信息
4. **上传附件**：点击"上传附件"按钮，可以上传与偏差相关的图片、文档等
5. **保存/提交**：点击"保存"按钮保存草稿，点击"提交"按钮提交偏差记录

### 3.2 偏差记录查询与筛选

1. **进入偏差列表**：在左侧菜单中点击"偏差管理"→"偏差记录"
2. **使用查询条件**：
   - 代码/标题：输入偏差代码或标题关键词
   - 状态：选择偏差状态进行筛选
   - 类型：选择偏差类型进行筛选
   - 严重程度：选择严重程度进行筛选
   - 日期范围：选择偏差创建或发生的日期范围
   - 报告人：选择偏差报告人进行筛选
3. **执行查询**：点击"查询"按钮执行筛选，点击"重置"按钮清除查询条件
4. **分页与排序**：使用页面底部的分页控件浏览多页结果，点击列标题进行排序

### 3.3 偏差详情查看

1. **选择偏差记录**：在偏差列表中点击任意一条偏差记录的标题或"查看"按钮
2. **查看偏差详情**：在详情页面可以查看偏差的所有信息，包括：
   - 基本信息：标题、描述、类型、严重程度等
   - 状态信息：当前状态、状态变更历史
   - 处理信息：调查、评估、纠正措施等
   - 附件列表：与偏差相关的附件
   - 评论与通知：相关评论和系统通知

### 3.4 偏差处理流程

#### 3.4.1 提交调查

1. **进入偏差详情**：在偏差列表中点击一条"新建"状态的偏差记录
2. **提交调查**：点击"提交调查"按钮
3. **指定调查人**：在弹出的对话框中选择调查人员
4. **确认提交**：点击"确认"按钮，系统将通知调查人员，偏差状态变为"调查中"

#### 3.4.2 偏差调查

1. **接收调查任务**：调查人员登录系统，收到待办任务通知
2. **进入偏差详情**：点击待办任务或直接进入偏差详情页面
3. **填写调查信息**：点击"填写调查"按钮，输入：
   - 调查结果：详细描述调查发现
   - 根本原因：分析并描述偏差的根本原因
   - 影响评估：评估偏差对产品质量、安全的影响
   - 建议措施：提出建议的纠正和预防措施
4. **提交调查结果**：点击"提交"按钮，偏差状态变为"待审批"

#### 3.4.3 偏差审批

1. **接收审批任务**：审批人员（通常是质量经理）登录系统，收到待办任务通知
2. **进入偏差详情**：点击待办任务或直接进入偏差详情页面
3. **审批操作**：
   - 审批通过：点击"批准"按钮，输入审批意见，指定负责人和截止日期
   - 驳回：点击"驳回"按钮，输入驳回理由，偏差退回给调查人员
4. **确认操作**：点击"确认"按钮，完成审批流程

#### 3.4.4 偏差关闭

1. **处理完成**：所有纠正措施执行完成后，负责人可以申请关闭偏差
2. **提交关闭申请**：点击"申请关闭"按钮，填写关闭报告
3. **审核关闭**：质量经理审核关闭报告，点击"关闭偏差"按钮
4. **确认关闭**：偏差状态变为"已关闭"，偏差流程结束

## 4. CAPA管理

### 4.1 CAPA计划创建

1. **进入CAPA管理模块**：在左侧菜单中点击"CAPA管理"→"CAPA计划"
2. **创建新CAPA**：点击页面右上角的"创建CAPA"按钮
3. **填写CAPA基本信息**：
   - 标题：输入CAPA的简要描述
   - 类型：选择CAPA类型（纠正措施、预防措施）
   - 来源：选择CAPA来源（偏差、审计发现、投诉、自检等）
   - 关联记录：选择关联的偏差、投诉等记录（可选）
   - 问题描述：详细描述需要解决的问题
   - 责任部门：选择负责执行CAPA的部门
   - 责任人：选择负责执行CAPA的人员
   - 目标完成日期：设置CAPA的目标完成日期
4. **添加措施计划**：点击"添加措施"按钮，填写各项具体措施
5. **保存/提交**：点击"保存"按钮保存草稿，点击"提交"按钮提交CAPA计划

### 4.2 CAPA措施执行

1. **接收CAPA任务**：CAPA责任人登录系统，收到待办任务通知
2. **进入CAPA详情**：点击待办任务或直接进入CAPA详情页面
3. **更新措施状态**：
   - 点击"更新状态"按钮
   - 选择措施当前状态（未开始、进行中、已完成、延期）
   - 填写执行情况和结果
   - 上传相关证据或附件（如需要）
4. **保存更新**：点击"保存"按钮，措施状态更新

### 4.3 CAPA有效性验证

1. **CAPA完成**：当所有措施都标记为"已完成"后，系统自动触发验证流程
2. **验证任务分配**：QA专员收到CAPA验证任务
3. **执行验证**：
   - 进入CAPA详情页面，点击"验证"按钮
   - 填写验证方法和过程
   - 记录验证结果和发现
   - 评估CAPA的有效性
4. **提交验证报告**：点击"提交"按钮，验证结果将提交给质量经理审核

### 4.4 CAPA关闭

1. **审核验证报告**：质量经理审核CAPA验证报告
2. **关闭操作**：
   - 验证有效：点击"关闭CAPA"按钮，CAPA状态变为"已关闭"
   - 验证无效：点击"重新执行"按钮，CAPA重新进入执行阶段
3. **归档记录**：CAPA关闭后，相关记录将被归档，可在历史记录中查询

## 5. 变更控制

### 5.1 变更申请创建

1. **进入变更控制模块**：在左侧菜单中点击"变更控制"→"变更申请"
2. **创建新变更**：点击页面右上角的"创建变更"按钮
3. **填写变更信息**：
   - 变更标题：输入变更的简要描述
   - 变更类型：选择变更类型（工艺变更、设备变更、文件变更、系统变更等）
   - 变更级别：选择变更级别（微小变更、一般变更、重要变更）
   - 变更描述：详细描述变更内容和背景
   - 影响范围：分析变更可能影响的范围
   - 提议人：填写变更提议人信息
   - 提议日期：选择变更提议日期
4. **添加附件**：上传与变更相关的支持文档
5. **保存/提交**：点击"保存"按钮保存草稿，点击"提交"按钮提交变更申请

### 5.2 变更评估与审批

1. **变更初审**：部门主管接收变更申请，进行初审
2. **变更评估**：质量部门对变更进行详细评估，确定变更影响
3. **变更审批**：根据变更级别，提交给相应级别的审批人进行审批
4. **审批结果**：
   - 批准：变更进入实施阶段
   - 有条件批准：变更在满足特定条件后进入实施阶段
   - 驳回：变更返回修改

### 5.3 变更实施

1. **实施计划制定**：变更负责人制定变更实施计划
2. **实施前检查**：实施前进行必要的检查和准备
3. **执行变更**：按照实施计划执行变更
4. **实施记录**：记录实施过程和结果
5. **完成确认**：相关人员确认变更实施完成

### 5.4 变更验证与关闭

1. **实施验证**：执行变更后的验证活动
2. **效果评估**：评估变更的效果和影响
3. **变更关闭**：验证通过后，变更可被正式关闭
4. **变更历史**：所有变更记录将被保存，可在历史记录中查询

## 6. 审计管理

### 6.1 审计计划创建

1. **进入审计管理模块**：在左侧菜单中点击"审计管理"→"审计计划"
2. **创建新审计**：点击页面右上角的"创建审计"按钮
3. **填写审计信息**：
   - 审计名称：输入审计的名称
   - 审计类型：选择审计类型（内部审计、供应商审计、客户审计、监管审计等）
   - 审计范围：指定审计的范围和对象
   - 审计日期：设置审计的开始和结束日期
   - 审计组成员：选择参与审计的人员
   - 审计标准：选择适用的审计标准
4. **保存/提交**：点击"保存"按钮保存草稿，点击"提交"按钮提交审计计划

### 6.2 审计执行

1. **审计准备**：审计组成员登录系统，查看审计计划和相关资料
2. **审计检查表**：系统根据审计类型自动生成审计检查表
3. **现场记录**：在审计过程中，记录发现的问题和观察结果
4. **上传证据**：上传审计过程中收集的证据和文档
5. **每日总结**：记录每日审计活动和发现

### 6.3 审计报告生成

1. **审计完成**：审计现场活动结束后
2. **生成报告**：点击"生成审计报告"按钮
3. **编辑报告内容**：
   - 审计概述：简要描述审计目的、范围和过程
   - 审计发现：详细记录所有发现项，按严重程度分类
   - 不符合项：列出需要整改的不符合项
   - 建议事项：提供改进建议
   - 结论：给出审计结论和总体评价
4. **审核报告**：审计负责人审核并批准审计报告
5. **发布报告**：点击"发布"按钮，报告将发送给相关人员

### 6.4 不符合项跟踪

1. **不符合项分配**：系统自动将不符合项分配给责任部门
2. **整改计划**：责任部门制定不符合项整改计划，包括纠正措施和完成日期
3. **整改跟踪**：QA专员定期跟踪整改进展
4. **整改验证**：整改完成后，QA专员进行验证
5. **关闭记录**：验证通过后，不符合项状态变为"已关闭"

## 7. 投诉管理

### 7.1 投诉记录创建

1. **进入投诉管理模块**：在左侧菜单中点击"投诉管理"→"投诉记录"
2. **创建新投诉**：点击页面右上角的"创建投诉"按钮
3. **填写投诉信息**：
   - 投诉编号：系统自动生成（可修改）
   - 投诉类型：选择投诉类型（产品质量、服务态度、交货延迟等）
   - 投诉来源：选择投诉来源（客户、内部员工、监管机构等）
   - 投诉人信息：填写投诉人姓名、联系方式等
   - 相关产品/批次：选择相关的产品和批次
   - 投诉内容：详细描述投诉内容
   - 投诉日期：选择投诉接收日期
   - 紧急程度：选择投诉的紧急程度（普通、紧急、特急）
4. **保存/提交**：点击"保存"按钮保存草稿，点击"提交"按钮提交投诉记录

### 7.2 投诉处理流程

1. **投诉受理**：投诉管理人员确认接收投诉
2. **投诉评估**：评估投诉的真实性和严重程度
3. **投诉调查**：对投诉内容进行调查，收集相关证据
4. **投诉处理**：制定并执行投诉处理方案
5. **客户沟通**：与投诉人进行沟通，反馈处理结果
6. **投诉关闭**：投诉处理完成后，进行投诉关闭操作

### 7.3 投诉统计与分析

1. **进入统计分析**：在左侧菜单中点击"投诉管理"→"投诉统计"
2. **选择统计维度**：
   - 按类型统计：查看不同类型投诉的数量和占比
   - 按时间统计：查看投诉在不同时间段的分布
   - 按产品统计：查看不同产品的投诉情况
   - 按处理结果统计：查看投诉的处理结果分布
3. **查看图表**：系统自动生成统计图表（柱状图、饼图、趋势图等）
4. **导出报告**：点击"导出"按钮，可将统计结果导出为Excel或PDF格式

## 8. 系统管理

### 8.1 用户管理

1. **进入用户管理**：在左侧菜单中点击"系统管理"→"用户管理"
2. **用户列表**：查看系统所有用户的基本信息
3. **创建用户**：点击"创建用户"按钮，填写用户信息，包括用户名、密码、所属部门、角色等
4. **编辑用户**：选择用户，点击"编辑"按钮，可以修改用户信息
5. **停用/启用用户**：选择用户，点击"停用"或"启用"按钮，控制用户账号状态
6. **重置密码**：选择用户，点击"重置密码"按钮，可以重置用户密码

### 8.2 角色权限管理

1. **进入角色管理**：在左侧菜单中点击"系统管理"→"角色管理"
2. **角色列表**：查看系统已定义的所有角色
3. **创建角色**：点击"创建角色"按钮，输入角色名称和描述
4. **分配权限**：选择角色，点击"分配权限"按钮，勾选该角色应有的权限
5. **编辑/删除角色**：对现有角色进行编辑或删除操作（系统预设角色不可删除）

### 8.3 系统设置

1. **进入系统设置**：在左侧菜单中点击"系统管理"→"系统设置"
2. **通用设置**：配置系统的通用参数，如日期格式、时间格式、分页大小等
3. **通知设置**：配置系统的通知方式，如邮件通知、站内消息等
4. **数据备份设置**：设置系统数据自动备份的频率和存储位置

### 8.4 日志查询

1. **进入日志查询**：在左侧菜单中点击"系统管理"→"日志查询"
2. **查询条件**：
   - 用户：选择特定用户
   - 操作类型：选择操作类型（登录、退出、创建、修改、删除等）
   - 时间范围：设置日志查询的时间范围
3. **执行查询**：点击"查询"按钮，查看符合条件的系统日志
4. **查看详情**：点击日志条目，可以查看日志的详细信息

## 9. 常见问题与解答

### 9.1 登录相关问题

**问题**：忘记密码怎么办？
**解答**：在登录页面点击"忘记密码"链接，输入用户名或邮箱，系统会发送密码重置链接到您的邮箱，按照提示操作即可重置密码。

**问题**：无法登录系统，提示账号被锁定。
**解答**：账号可能因多次登录失败被系统自动锁定。请联系系统管理员解锁账号，或者等待锁定时间过期（通常为30分钟）。

### 9.2 功能操作问题

**问题**：如何查看我的待办任务？
**解答**：登录系统后，在首页的"待办任务"区域可以查看分配给您的所有待办任务，点击任务标题可以直接进入相关操作页面。

**问题**：如何上传大文件作为附件？
**解答**：系统支持最大100MB的文件上传。如果文件过大，请先压缩文件，或者分多个小文件上传。

**问题**：如何导出数据报表？
