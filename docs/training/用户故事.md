## 角色定义

| 角色名称 | 职责描述 |
|--------|--------|
| **系统管理员** | 负责系统的整体管理，包括用户管理、权限配置、系统配置等 |
| **培训管理员** | 负责培训需求管理、培训计划制定、培训执行跟踪等 |
| **培训讲师** | 创建和管理培训课程，执行培训活动，评估培训效果 |
| **部门经理** | 审核部门培训需求，批准培训计划，查看部门培训报表 |
| **普通员工** | 查看个人培训计划，报名参加培训，完成培训评估等 |
| **质量管理员** | 监督培训管理流程的合规性，进行培训审计等 |
| **HR管理员** | 管理员工基本信息，与HR系统集成，参与培训计划制定 |
| **管理层** | 查看企业整体培训报表，制定培训战略，分配培训资源 |

## 培训需求管理

### 需求识别功能

#### USER-STORY-001: 提交培训需求
- **标题**: 作为部门经理，我希望能够提交部门培训需求，以便满足部门员工的技能提升需求
- **角色**: 部门经理
- **优先级**: 高
- **描述**: 部门经理能够登录系统，选择培训需求提交功能，填写需求详细信息，包括需求名称、培训目标、培训对象、期望时间等，并提交审批
- **验收标准**:
  1. 部门经理能够成功登录系统并访问培训需求提交页面
  2. 能够完整填写培训需求表单并提交
  3. 系统能够正确保存培训需求信息
  4. 提交后，培训需求状态变为"待审核"

#### USER-STORY-002: 查看培训需求列表
- **标题**: 作为培训管理员，我希望能够查看所有培训需求的列表，以便进行需求评估和管理
- **角色**: 培训管理员
- **优先级**: 高
- **描述**: 培训管理员登录系统后，能够在培训需求管理模块查看所有培训需求的列表，包括需求名称、提交人、提交时间、状态等信息，并支持按状态、部门等条件筛选
- **验收标准**:
  1. 培训管理员能够访问培训需求列表页面
  2. 页面显示所有培训需求的基本信息
  3. 能够使用筛选条件过滤需求列表
  4. 能够点击需求查看详细信息

### 需求评估与审批

#### USER-STORY-003: 评估培训需求
- **标题**: 作为培训管理员，我希望能够评估提交的培训需求，以便确定其必要性和优先级
- **角色**: 培训管理员
- **优先级**: 高
- **描述**: 培训管理员能够查看待评估的培训需求详情，根据需求的重要性、紧急性、成本效益等因素进行评估，填写评估意见和建议，并确定优先级
- **验收标准**:
  1. 能够打开待评估的培训需求详情
  2. 能够填写评估意见和建议
  3. 能够设置需求优先级（高、中、低）
  4. 保存评估后，需求状态更新为"已评估"

#### USER-STORY-004: 审批培训需求
- **标题**: 作为培训总监，我希望能够审批培训需求，以便控制培训资源的合理分配
- **角色**: 培训管理员（代表培训总监角色）
- **优先级**: 高
- **描述**: 培训总监能够查看已评估的培训需求，根据企业战略、预算情况等因素决定是否批准需求，并提供审批意见
- **验收标准**:
  1. 能够查看待审批的培训需求列表
  2. 能够查看需求的详细信息和评估意见
  3. 能够执行批准或拒绝操作
  4. 能够填写审批意见
  5. 审批后，需求状态更新为"已批准"或"已拒绝"

## 培训计划制定

### 计划创建与管理

#### USER-STORY-005: 创建年度培训计划
- **标题**: 作为培训管理员，我希望能够创建年度培训计划，以便系统地规划全年培训活动
- **角色**: 培训管理员
- **优先级**: 高
- **描述**: 培训管理员能够基于已批准的培训需求，创建年度培训计划，设置计划名称、年度、包含的课程、预算等信息
- **验收标准**:
  1. 能够进入培训计划创建页面
  2. 能够设置计划基本信息（名称、年度等）
  3. 能够从已批准的需求中选择要纳入计划的项目
  4. 能够保存计划并生成计划编号

#### USER-STORY-006: 查看个人培训计划
- **标题**: 作为普通员工，我希望能够查看我的个人培训计划，以便了解我需要参加的培训活动
- **角色**: 普通员工
- **优先级**: 高
- **描述**: 员工登录系统后，能够在个人培训模块查看分配给自己的培训计划，包括培训课程、时间、地点等信息
- **验收标准**:
  1. 员工能够访问个人培训计划页面
  2. 页面显示所有分配给该员工的培训活动
  3. 能够查看每个培训活动的详细信息
  4. 能够按时间顺序或状态筛选培训活动

### 计划审批与发布

#### USER-STORY-007: 审批培训计划
- **标题**: 作为部门经理，我希望能够审批本部门的培训计划，以便确保计划符合部门工作安排
- **角色**: 部门经理
- **优先级**: 中
- **描述**: 部门经理能够查看待审批的部门培训计划，了解计划内容和资源需求，并决定是否批准该计划
- **验收标准**:
  1. 能够查看待审批的部门培训计划
  2. 能够查看计划的详细内容和资源需求
  3. 能够执行批准或拒绝操作
  4. 审批后，计划状态更新

- **优先级**: 高

- **验收标准**:
  1. 能够进入课程创建页面
  2. 能够填写课程的基本信息和详细内容
  3. 能够上传培训资料（如PPT、文档等）
  4. 能够保存课程并生成课程编号

#### USER-STORY-009: 管理课程资料
- **标题**: 作为培训讲师，我希望能够管理我的课程资料，以便及时更新培训内容
- **角色**: 培训讲师
- **优先级**: 中
- **描述**: 培训讲师能够查看、修改和删除自己创建的课程资料，上传新版本的资料，并查看资料的使用情况
- **验收标准**:
  1. 能够查看已上传的课程资料列表
  2. 能够上传新的课程资料
  3. 能够删除或替换现有资料
  4. 系统记录资料的版本历史

- **优先级**: 高

#### USER-STORY-011: 审核培训报名
- **标题**: 作为培训管理员，我希望能够审核员工的培训报名申请，以便控制培训人数和质量

- **优先级**: 中

- **验收标准**:
  1. 能够查看待审核的报名申请列表
  2. 能够查看申请人信息和相关背景
  3. 能够执行批准或拒绝操作

### 培训签到与出勤

#### USER-STORY-012: 电子签到

- **角色**: 培训管理员
- **优先级**: 高
- **描述**: 培训管理员能够在培训开始前生成签到二维码，参会员工通过扫描二维码或系统内签到功能完成签到
- **验收标准**:
  1. 能够为培训活动生成唯一的签到二维码
  2. 员工能够通过扫描二维码完成签到
  3. 签到数据自动记录到系统中
  4. 能够实时查看签到人数和签到状态

- **描述**: 员工能够在培训结束后提交培训反馈，包括对课程内容、讲师表现、培训组织等方面的评价和建议

### 证书到期提醒

#### USER-STORY-018: 接收证书到期提醒
- **标题**: 作为普通员工，我希望能够收到证书到期提醒，以便及时安排复训
- **角色**: 普通员工
- **优先级**: 高
- **描述**: 当员工的培训证书即将到期时，系统能够通过邮件、短信或系统消息提醒员工及时参加复训

  1. 当证书即将到期（如30天内）时，系统自动触发提醒
  2. 员工能够通过至少一种渠道（邮件、短信或系统消息）收到提醒
  3. 提醒内容包括证书名称、到期日期和复训建议

## 附录

### 术语定义
- **GMP**: 药品生产质量管理规范(Good Manufacturing Practice)
- **培训需求**: 员工或部门为履行职责所需的知识和技能的缺口
- **培训计划**: 规定何时、何地、由谁进行何种培训的安排
- **培训记录**: 记录培训活动、参与者、结果等信息的文档
- **能力矩阵**: 展示员工在不同技能领域的能力水平的矩阵
- **证书**: 证明员工完成特定培训的正式文件
- **资质**: 员工具备的专业技能和认证
- **复训**: 定期重复进行的培训，用于更新知识和技能
- **有效性评估**: 评估培训是否达到预期目标的过程
- **合规性**: 符合相关法规和标准的状态
- **审计追踪**: 记录系统中所有操作的过程，用于合规性和安全管理
- **电子签名**: 符合法规要求的电子形式的签名，用于确认电子记录的真实性
- **用户故事**: 描述系统功能的简短叙述，格式为"作为...我希望...以便..."
- **验收标准**: 确定功能是否满足需求的具体条件和标准
