## 1️⃣ 手册说明

### 1.1 手册目的

本文档作为GMP环境下人事管理子系统的培训手册，旨在为系统用户提供全面的操作指导和参考。通过本手册，用户将了解系统的基本概念、功能模块、操作流程以及GMP合规性要求，从而能够正确、高效地使用系统，确保系统在GMP环境下的合规运行。

### 1.2 适用对象

本培训手册适用于以下人员：
- **系统管理员**：负责系统管理和维护的技术人员
- **终端用户**：直接使用系统进行日常操作的人员，包括人事部门人员、部门经理等
- **IT支持人员**：提供系统支持和故障排除的技术人员
- **质量管理人员**：关注系统GMP合规性的质量人员
- **管理层**：需要了解系统功能和价值的管理人员

### 1.3 手册使用指南

- 新手用户建议按照手册的章节顺序进行学习，从系统概述开始，逐步深入了解系统功能
- 有经验的用户可以直接查阅需要的章节，作为操作参考
- 手册中的**注意事项**和**警告信息**需要特别关注，确保GMP合规性和操作安全
- 手册中的操作步骤均配有详细说明，用户可以按照步骤进行实践操作
- 手册中的术语解释部分提供了系统常用术语的定义，方便用户理解

## 2️⃣ 系统概述

### 2.1 系统目标与价值

**系统目标**：
- 建立符合GMP要求的人事管理信息系统
- 实现人事管理流程的标准化、规范化和自动化
- 确保人员数据的完整性、准确性和可追溯性
- 支持质量体系的有效运行和持续改进
- 提高人事管理效率和决策支持能力

**系统价值**：
- **合规性保障**：满足GMP法规对人员管理的要求，提供完整的审计跟踪
- **效率提升**：自动化处理人事业务，减少手动操作和错误
- **数据质量**：确保人员数据的准确性和一致性，支持数据分析
- **风险管理**：通过权限管理和审计功能，降低合规风险
- **决策支持**：提供全面的报表和分析功能，支持管理层决策

### 2.2 系统架构

**系统架构概述**：
人事管理子系统采用多层架构设计，包括：
- **前端展示层**：用户界面，提供交互操作
- **业务逻辑层**：处理业务逻辑，实现系统功能
- **数据访问层**：负责数据存取和管理
- **数据库层**：存储系统数据
- **集成层**：与其他系统进行集成

**技术架构**：
- 前端：基于现代Web技术，支持响应式设计
- 后端：企业级应用框架
- 数据库：关系型数据库，支持事务和数据完整性
- 安全：采用多层安全机制，包括身份认证、授权、加密等

### 2.4 GMP合规性设计

**数据完整性保障**：
- 采用事务机制确保数据一致性
- 实施数据校验和验证规则
- 防止未授权的数据修改
- 确保数据的准确性和完整性

**审计跟踪功能**：
- 记录所有关键操作和数据变更
- 跟踪用户登录和活动
- 保存操作时间、操作人、操作内容等信息
- 审计日志不可篡改和删除

**电子签名功能**：
- 符合21 CFR Part 11的电子签名要求
- 支持多级审批和签名
- 确保签名的唯一性和不可否认性
- 记录签名时间和理由

**权限管理**：
- 基于角色的访问控制
- 最小权限原则
- 权限分配和审批流程
- 定期权限审核

## 3️⃣ 系统登录与基本操作

### 3.1 系统登录

**登录步骤**：
1. 打开浏览器，输入系统URL：`https://gmp-hr.company.com`
2. 系统显示登录页面，输入用户名和密码
3. 点击【登录】按钮
4. 首次登录或密码过期时，系统会提示修改密码
5. 密码修改成功后，进入系统主界面

**登录注意事项**：
- 用户名和密码区分大小写
- 密码必须符合复杂度要求（至少8位，包含字母、数字和特殊字符）
- 连续3次登录失败，账户将被锁定，需联系系统管理员解锁
- 登录后系统将记录登录日志，包括登录时间、IP地址等信息

**电子签名登录（适用于关键操作）**：
1. 完成普通登录后，在执行关键操作时，系统会提示进行电子签名
2. 输入用户名、密码和签名理由
3. 点击【确认签名】按钮
4. 签名成功后，系统会记录签名信息并执行操作

### 3.2 系统界面导航

**界面布局**：
- **顶部导航栏**：显示系统名称、用户信息、通知、帮助等
- **左侧菜单栏**：系统功能模块导航
- **主内容区**：显示当前功能的详细内容
- **右侧工具栏**：常用操作和快捷功能

**导航操作**：
- 点击左侧菜单栏的模块名称，进入相应功能模块
- 使用顶部导航栏的搜索功能，快速查找内容
- 通过面包屑导航，了解当前页面位置
- 使用顶部导航栏的用户菜单，访问个人设置、退出登录等

**个性化设置**：
- 点击顶部导航栏的用户头像，选择【个人设置】
- 在个人设置页面，可以修改密码、调整界面偏好、设置通知方式等
- 保存设置后，系统会应用新的配置

### 3.3 基本操作规范

**数据录入规范**：
- 严格按照字段要求输入数据，确保准确性
- 必填字段必须填写完整
- 日期、数字等格式必须符合系统要求
- 避免输入特殊字符或超过字段长度的内容

**数据查询规范**：
- 使用系统提供的查询条件进行精准查询
- 查询结果可导出为Excel、PDF等格式
- 高级查询可设置多个条件组合查询
- 查询历史记录可保存和重复使用

**操作流程规范**：
- 按照系统预设的工作流程进行操作
- 关键操作需要进行电子签名确认
- 审批操作需要查看完整信息后再进行决策
- 操作完成后及时保存并退出

**GMP合规性注意事项**：
- 所有操作必须在系统中记录，不得离线操作
- 确保电子签名的真实性和唯一性
- 定期查看系统通知和消息
- 及时报告系统异常和问题

## 4️⃣ 员工信息管理模块

### 4.1 员工档案管理

**功能概述**：
员工档案管理模块用于维护员工的基本信息、工作经历、教育背景等全面的个人档案信息，确保员工信息的完整性和准确性。

**主要功能**：
- 员工信息录入和编辑
- 员工档案查询和导出
- 员工状态管理（在职、离职、休假等）
- 员工照片和证件上传
- 档案变更历史记录

**操作步骤**：

**新增员工**：
1. 点击左侧菜单【员工管理】→【员工档案】
2. 点击页面上方的【新增员工】按钮
3. 填写员工基本信息（姓名、性别、身份证号、联系方式等）
4. 填写工作信息（部门、岗位、入职日期、薪资等级等）
5. 上传员工照片和相关证件
6. 点击【保存】按钮
7. 系统提示保存成功，并显示员工详情页面

**编辑员工信息**：
1. 在员工档案列表页面，找到需要编辑的员工
2. 点击操作列的【编辑】按钮
3. 修改员工信息
4. 点击【保存】按钮
5. 系统提示保存成功

**查询员工信息**：
1. 在员工档案列表页面，使用查询条件（姓名、部门、工号等）
2. 输入查询值，点击【查询】按钮
3. 系统显示符合条件的员工列表
4. 可使用【重置】按钮清空查询条件

**员工状态变更**：
1. 在员工档案列表页面，找到需要变更状态的员工
2. 点击操作列的【状态变更】按钮
3. 选择新状态（在职、离职、休假、调岗等）
4. 填写变更原因和生效日期
5. 进行电子签名确认
6. 点击【确认变更】按钮
7. 系统提示状态变更成功

**GMP合规性要求**：
- 员工身份证号、教育背景等关键信息必须真实准确
- 员工状态变更必须记录变更原因和审批信息
- 员工档案修改必须保留历史记录，确保可追溯性
- 员工敏感信息访问必须符合权限要求

### 4.2 人员资质管理

**功能概述**：
人员资质管理模块用于管理员工的专业资质、证书、技能等信息，确保员工具备岗位所需的资质要求，满足GMP对人员资质的管理要求。

**主要功能**：
- 资质信息录入和管理
- 证书上传和有效期管理
- 资质到期提醒
- 资质查询和统计
- 资质变更历史记录

**操作步骤**：

**新增资质**：
1. 点击左侧菜单【员工管理】→【人员资质】
2. 选择员工，点击【新增资质】按钮
3. 填写资质信息（资质名称、发证机构、获得日期、有效期等）
4. 上传资质证书扫描件
5. 设置提醒时间
6. 点击【保存】按钮

**资质更新**：
1. 在人员资质列表页面，找到需要更新的资质
2. 点击操作列的【更新】按钮
3. 修改资质信息
4. 点击【保存】按钮

**资质查询**：
1. 在人员资质列表页面，使用查询条件（员工姓名、资质名称、状态等）
2. 输入查询值，点击【查询】按钮
3. 系统显示符合条件的资质列表

**资质到期提醒**：
1. 系统会自动识别即将到期的资质（默认提前30天）
2. 在首页和通知中心显示资质到期提醒
3. 管理员可以查看资质到期报表

**GMP合规性要求**：
- 关键岗位人员必须具备相应的资质和证书
- 资质证书必须在有效期内，过期必须及时更新
- 资质信息变更必须记录历史，确保可追溯性
- 定期对人员资质进行审核和验证

### 4.3 员工培训记录管理

**功能概述**：
员工培训记录管理模块用于记录和跟踪员工的培训情况，确保员工接受必要的培训，满足GMP对人员培训的管理要求。

**主要功能**：
- 培训记录录入和管理
- 培训成绩和评估记录
- 培训证书管理
- 培训需求分析
- 培训计划制定和执行

**操作步骤**：

**新增培训记录**：
1. 点击左侧菜单【员工管理】→【培训记录】
2. 选择员工，点击【新增培训记录】按钮
3. 填写培训信息（培训名称、培训日期、培训内容、培训时长等）
4. 填写培训成绩和评估结果
5. 上传培训证书或证明材料
6. 点击【保存】按钮

**培训记录查询**：
1. 在培训记录列表页面，使用查询条件（员工姓名、培训名称、日期范围等）
2. 输入查询值，点击【查询】按钮
3. 系统显示符合条件的培训记录列表

**培训需求分析**：
1. 点击左侧菜单【培训管理】→【需求分析】
2. 系统显示培训需求分析报表
3. 可按部门、岗位、人员等维度进行分析

**GMP合规性要求**：
- 所有员工必须接受GMP相关培训
- 培训记录必须真实完整，包括培训内容、时间、成绩等
- 关键岗位人员必须定期接受复训
- 培训记录必须保存完整，确保可追溯性

## 5️⃣ 考勤管理模块

### 5.1 考勤记录管理

**功能概述**：
考勤记录管理模块用于记录和管理员工的出勤情况，包括打卡记录、出勤统计等，确保考勤数据的准确性和可追溯性。

**主要功能**：
- 打卡记录查询和管理
- 考勤异常处理
- 考勤统计和报表
- 考勤规则设置
- 与门禁或打卡系统集成

**操作步骤**：

**考勤记录查询**：
1. 点击左侧菜单【考勤管理】→【考勤记录】
2. 选择查询条件（员工姓名、部门、日期范围等）
3. 点击【查询】按钮
4. 系统显示符合条件的考勤记录

**考勤异常处理**：
1. 在考勤记录列表页面，系统标记显示异常记录（迟到、早退、旷工等）
2. 点击操作列的【处理】按钮
3. 根据实际情况进行补卡、请假或其他处理
4. 填写处理原因
5. 进行电子签名确认
6. 点击【确认处理】按钮

**考勤统计**：
1. 点击左侧菜单【考勤管理】→【考勤统计】
2. 选择统计条件（部门、月份等）
3. 点击【生成报表】按钮
4. 系统显示考勤统计报表
5. 可导出为Excel或PDF格式

**GMP合规性要求**：
- 考勤记录必须真实准确，不得篡改
- 考勤异常处理必须有审批记录
- 关键岗位的出勤情况必须严格监控
- 考勤数据必须定期备份，确保数据安全

排班管理模块用于制定和管理员工的排班计划，确保工作的正常进行，满足GMP对人员配置的管理要求。

**主要功能**：
- 排班计划制定
- 班次管理
- 排班调整和申请
- 排班冲突检测
- 排班统计和报表

2. 选择月份和部门
3. 点击【制定排班】按钮
4. 为员工分配班次
5. 系统自动检测排班冲突
6. 点击【保存排班】按钮

**排班调整**：
1. 在排班管理页面，找到需要调整的排班
2. 点击操作列的【调整】按钮
3. 修改班次安排
4. 填写调整原因
5. 进行电子签名确认
6. 点击【确认调整】按钮

**排班查询**：
1. 在排班管理页面，使用查询条件（员工姓名、部门、日期等）
2. 输入查询值，点击【查询】按钮
3. 系统显示符合条件的排班信息

**GMP合规性要求**：
- 关键岗位必须保证足够的人员配置
- 排班调整必须记录原因和审批信息
- 特殊岗位的人员排班必须符合法规要求
- 排班数据必须可追溯，便于审计

**主要功能**：
- 假期类型设置
- 请假申请和审批
- 假期余额管理
- 假期统计和报表
- 与考勤系统集成

**操作步骤**：

**提交请假申请**：
1. 点击左侧菜单【考勤管理】→【假期管理】

## 🔟 常见问题与故障排除

### 10.1 登录问题

**问题1：无法登录系统**
- **可能原因**：用户名或密码错误
- **解决方法**：检查用户名和密码是否正确，注意区分大小写
- **注意事项**：连续3次登录失败，账户将被锁定

**问题2：账户被锁定**
- **可能原因**：多次密码错误或安全策略触发
- **解决方法**：联系系统管理员解锁账户
- **注意事项**：解锁后请及时修改密码

**问题3：密码忘记**
- **可能原因**：用户忘记密码
- **解决方法**：点击登录页面的【忘记密码】，按照提示重置密码
- **注意事项**：密码重置需要通过邮箱或手机验证

### 10.2 功能操作问题

**问题1：无法添加或修改数据**
- **可能原因**：权限不足或数据格式错误
- **解决方法**：检查用户权限，确保数据格式符合要求
- **注意事项**：必填字段必须填写完整

**问题2：查询结果不准确**
- **可能原因**：查询条件设置错误或数据更新不及时
- **解决方法**：检查查询条件，尝试刷新页面
- **注意事项**：复杂查询可分步进行

**问题3：审批流程卡住**
- **可能原因**：审批人未处理或工作流配置错误
- **解决方法**：联系相关审批人，或通知系统管理员检查工作流配置
- **注意事项**：紧急情况可申请流程调整

### 10.3 系统性能问题

**问题1：系统运行缓慢**
- **可能原因**：网络问题、服务器负载高或浏览器缓存问题
- **解决方法**：检查网络连接，清除浏览器缓存，或联系IT支持
- **注意事项**：避免同时打开多个大型报表

**问题2：页面加载失败**
- **可能原因**：网络中断或服务器问题
- **解决方法**：检查网络连接，刷新页面，或等待服务器恢复
- **注意事项**：重要操作前保存数据

**问题3：报表生成超时**
- **可能原因**：数据量过大或查询条件复杂
- **解决方法**：优化查询条件，减少数据量，或使用分页查询
- **注意事项**：大数据报表可选择异步生成

### 10.4 GMP合规性问题

**问题1：电子签名失败**
- **可能原因**：签名权限不足或签名信息不完整
- **解决方法**：检查签名权限，确保填写完整的签名理由
- **注意事项**：电子签名必须由本人进行，不得代签

**问题2：审计日志查询不到**
- **可能原因**：权限不足或查询条件设置错误
- **解决方法**：检查用户权限，调整查询条件
- **注意事项**：审计日志查询有专门的权限控制

**问题3：数据完整性校验失败**
- **可能原因**：数据缺失或数据格式错误
- **解决方法**：检查数据完整性，修正错误数据
- **注意事项**：关键数据必须符合GMP要求
