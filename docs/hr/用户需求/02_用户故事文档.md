# 人力资源管理子系统（HRMS）用户故事文档

## 1. 简介

### 1.1 文档目的
本文档通过用户故事的形式，从用户视角描述人力资源管理子系统的功能需求，帮助团队更好地理解用户需求，指导开发和测试工作。

### 1.2 编写规范
- 用户故事格式：作为[用户角色]，我希望[功能描述]，以便[业务价值]
- 验收标准：明确、可验证的功能实现标准
- 优先级：使用MoSCoW方法划分（Must have、Should have、Could have、Won't have）

## 2. 组织架构管理用户故事

### 2.1 部门管理

**用户故事ID**: US-HR-ORG-001
**用户角色**: HR管理人员
**用户故事**: 作为HR管理人员，我希望能够创建、修改和删除部门，以便维护最新的组织结构。
**验收标准**:
- 能够创建多级部门结构
- 能够修改部门名称、负责人和描述信息
- 能够删除不再使用的部门（无员工的部门）
- 部门移动时，其下的子部门和员工能够正确跟随
**优先级**: Must have

**用户故事ID**: US-HR-ORG-002
**用户角色**: HR管理人员
**用户故事**: 作为HR管理人员，我希望能够查看部门结构树，以便清晰了解整个组织架构。
**验收标准**:
- 系统能够以树形结构展示所有部门
- 能够展开/折叠部门树节点
- 鼠标悬停时显示部门基本信息
- 支持搜索定位特定部门
**优先级**: Must have

### 2.2 岗位管理

**用户故事ID**: US-HR-ORG-003
**用户角色**: HR管理人员
**用户故事**: 作为HR管理人员，我希望能够创建和管理岗位信息，以便明确各岗位的职责和要求。
**验收标准**:
- 能够创建新岗位，设置岗位名称、所属部门、职责描述等
- 能够修改现有岗位信息
- 能够为岗位分配编制人数
- 能够查看岗位对应的员工列表
**优先级**: Must have

**用户故事ID**: US-HR-ORG-004
**用户角色**: 部门主管
**用户故事**: 作为部门主管，我希望能够查看本部门的岗位设置，以便了解部门岗位配置情况。
**验收标准**:
- 能够查看部门下所有岗位的基本信息
- 能够查看岗位的编制和实际人数对比
- 能够查看岗位的职责描述和任职要求
**优先级**: Should have

## 3. 员工档案管理用户故事

### 3.1 员工信息管理

**用户故事ID**: US-HR-EMP-001
**用户角色**: HR管理人员
**用户故事**: 作为HR管理人员，我希望能够录入新员工信息并创建员工档案，以便开始员工的管理流程。
**验收标准**:
- 能够录入员工基本信息（姓名、性别、身份证号、联系方式等）
- 能够上传员工照片和证件扫描件
- 系统自动生成员工工号
- 能够关联员工与认证系统的用户账号
**优先级**: Must have

**用户故事ID**: US-HR-EMP-002
**用户角色**: HR管理人员
**用户故事**: 作为HR管理人员，我希望能够修改员工信息并记录变更历史，以便确保员工信息的准确性和可追溯性。
**验收标准**:
- 能够修改员工的各类信息
- 系统自动记录每次变更的内容、时间和操作人
- 能够查看员工信息的完整变更历史
- 敏感信息变更需要二次确认
**优先级**: Must have

**用户故事ID**: US-HR-EMP-003
**用户角色**: 普通员工
**用户故事**: 作为普通员工，我希望能够查看和修改个人部分信息，以便保持信息的及时性。
**验收标准**:
- 能够查看本人的详细档案信息
- 能够修改联系方式、紧急联系人等非关键信息
- 不能修改身份信息、入职日期等关键信息
- 修改操作有日志记录
**优先级**: Should have

### 3.2 合同管理

**用户故事ID**: US-HR-EMP-004
**用户角色**: HR管理人员
**用户故事**: 作为HR管理人员，我希望能够管理员工的劳动合同信息，以便跟踪合同状态和到期情况。
**验收标准**:
- 能够录入劳动合同信息（签订日期、到期日期、合同类型等）
- 能够上传合同扫描件
- 系统自动提醒即将到期的合同
- 能够记录合同续签信息
**优先级**: Must have

- 能够查看待审批的考勤异常申请
- 能够审批通过或拒绝申请

- 审批结果自动更新考勤记录

## 6. 资质认证管理用户故事

### 6.1 资质管理

**用户故事ID**: US-HR-QUL-001
**用户角色**: HR管理人员
**用户故事**: 作为HR管理人员，我希望能够管理员工的各类资质证书，以便确保员工具备岗位所需的专业能力。
**验收标准**:
- 能够定义不同类型的资质证书
- 能够录入员工获得的资质信息（证书名称、颁发机构、有效期等）
- 能够上传资质证书扫描件
- 能够关联资质与岗位要求
**优先级**: Must have

**用户故事ID**: US-HR-QUL-002
**用户角色**: HR管理人员
**用户故事**: 作为HR管理人员，我希望系统能够自动提醒资质证书即将到期，以便及时安排复审或重新考核。
**验收标准**:
- 系统能够根据设置的预警天数，提前提醒资质到期
- 能够批量处理即将到期的资质
- 能够生成资质到期预警报表
- 预警信息能够通过邮件或系统消息发送
**优先级**: Must have

### 6.2 资质与权限关联

**用户故事ID**: US-HR-QUL-003
**用户角色**: 系统管理员
**用户故事**: 作为系统管理员，我希望能够将员工资质与系统操作权限关联，以便确保员工只能执行其具备资质的操作。
**验收标准**:
- 能够设置资质与操作权限的映射关系
- 当员工获得或失去相关资质时，系统自动更新其权限
- 能够查看员工资质与权限的对应关系
- 权限变更有审计日志记录
**优先级**: Must have

## 9. 系统集成用户故事

### 9.1 与认证系统集成

**用户故事ID**: US-HR-INT-001
**用户角色**: HR管理人员
**用户故事**: 作为HR管理人员，我希望在创建员工档案时能够自动在认证系统中创建用户账号，以便员工能够快速访问系统。
**验收标准**:
- 创建员工档案时，系统自动在认证系统中创建对应用户账号
- 员工基本信息变更时，认证系统用户信息同步更新
- 员工离职时，能够自动禁用认证系统中的用户账号
- 能够查看员工档案与认证系统账号的关联状态
**优先级**: Must have

### 9.2 与生产系统集成

**用户故事ID**: US-HR-INT-002
**用户角色**: 生产管理人员
**用户故事**: 作为生产管理人员，我希望生产系统能够验证操作人员的资质，以便确保只有具备相应资质的人员才能进行特定操作。
**验收标准**:
- 生产系统能够实时查询员工资质信息
- 当员工不具备操作资质时，系统拒绝其进行相关操作
- 资质过期的员工操作会触发警告
- 操作记录包含操作人员的资质验证信息
**优先级**: Must have
