## 1️⃣ 手册说明

### 1.1 文档目的

本文档作为GMP环境下人事管理子系统实施的操作手册，旨在为实施团队提供详细的实施步骤、操作指导和注意事项。通过本手册，实施人员可以全面了解系统实施的各个环节，确保系统能够顺利部署、配置和上线，并满足GMP合规性要求。

### 1.2 适用范围

本手册适用于参与人事子系统实施的技术人员，包括但不限于：

- 系统实施工程师
- 数据库管理员
- 系统配置人员
- 测试人员
- 技术支持人员

### 1.3 术语定义

| 术语 | 解释 |
|------|------|
| GMP | 良好制造规范（Good Manufacturing Practice），确保产品质量和安全的质量体系 |
| 人事子系统 | GMP系统中的模块，负责员工信息、考勤、薪资、绩效、培训等人事相关管理功能 |
| 实施环境 | 系统部署和运行的硬件、软件和网络环境 |
| 数据迁移 | 将现有系统中的数据转移到新系统的过程 |
| 系统配置 | 根据业务需求对系统功能和参数进行设置的过程 |
| 接口集成 | 实现不同系统之间数据交换和功能调用的过程 |
| 电子记录 | 通过电子方式创建、修改、维护、存档、检索和分发的记录 |
| 审计跟踪 | 记录系统中所有关键操作的详细信息，包括操作者、操作时间、操作内容等 |
| 用户权限 | 控制用户对系统功能和数据的访问权限 |

### 2.3 网络环境准备

#### 2.3.1 网络拓扑设计

1. **核心网络架构**：
   - 采用三层网络架构：核心层、汇聚层、接入层
   - 核心交换机连接所有服务器和汇聚交换机
   - 汇聚交换机连接各部门的接入交换机
   - 接入交换机连接用户终端

2. **网络安全设计**：
   - 部署防火墙保护核心服务器
   - 配置网络隔离区（DMZ）用于外部系统对接
   - 实施网络访问控制列表（ACL）限制访问
   - 配置VPN服务支持远程安全访问

#### 2.3.2 网络参数配置

1. **IP地址规划**：

| 网络区域 | IP地址范围 | 子网掩码 | 网关地址 | 备注 |

| 客户端区域 | 192.168.2.0/24 | 255.255.255.0 | 192.168.2.1 | 用于办公区域终端 |

## 3️⃣ 系统安装

### 3.1 数据库安装

#### 3.1.1 Oracle数据库安装步骤

1. **安装前准备**：
   - 以root用户登录Linux服务器
   - 创建oracle用户和oinstall、dba组
   - 配置内核参数（在/etc/sysctl.conf中）
   - 配置资源限制（在/etc/security/limits.conf中）
   - 创建Oracle安装目录和数据目录
   - 设置oracle用户环境变量

2. **安装Oracle软件**：
   - 以oracle用户登录服务器
   - 解压Oracle安装包
   - 运行安装程序：`./runInstaller`
   - 选择"创建和配置数据库"选项
   - 选择"服务器类"安装类型
   - 选择"单实例数据库安装"选项
   - 选择数据库版本（企业版）
   - 指定Oracle主目录和软件位置
   - 选择数据库类型（数据仓库）
   - 设置全局数据库名和SID
   - 配置字符集（AL32UTF8）
   - 设置数据库管理员密码
   - 配置数据库存储位置
   - 配置闪回恢复区
   - 配置数据库选项和参数
   - 执行先决条件检查
   - 确认安装配置并开始安装
   - 根据提示执行root.sh脚本

3. **安装后配置**：
   - 启动监听器：`lsnrctl start`
   - 启动数据库实例：`sqlplus / as sysdba`，然后执行`startup`
   - 验证数据库连接：`sqlplus sys/password as sysdba`
   - 配置自动启动服务（在/etc/oratab中设置）

#### 3.1.2 数据库初始化配置

1. **创建表空间**：

```sql
-- 创建系统表空间
CREATE TABLESPACE SYSTEM_DATA DATAFILE '/u01/app/oracle/oradata/HRDB/system_data01.dbf' SIZE 500M AUTOEXTEND ON NEXT 100M MAXSIZE UNLIMITED LOGGING EXTENT MANAGEMENT LOCAL SEGMENT SPACE MANAGEMENT AUTO;

-- 创建用户表空间
CREATE TABLESPACE USER_DATA DATAFILE '/u01/app/oracle/oradata/HRDB/user_data01.dbf' SIZE 1000M AUTOEXTEND ON NEXT 200M MAXSIZE UNLIMITED LOGGING EXTENT MANAGEMENT LOCAL SEGMENT SPACE MANAGEMENT AUTO;

-- 创建索引表空间
CREATE TABLESPACE INDEX_DATA DATAFILE '/u01/app/oracle/oradata/HRDB/index_data01.dbf' SIZE 500M AUTOEXTEND ON NEXT 100M MAXSIZE UNLIMITED LOGGING EXTENT MANAGEMENT LOCAL SEGMENT SPACE MANAGEMENT AUTO;

-- 创建临时表空间
CREATE TEMPORARY TABLESPACE TEMP_DATA TEMPFILE '/u01/app/oracle/oradata/HRDB/temp_data01.dbf' SIZE 200M AUTOEXTEND ON NEXT 50M MAXSIZE UNLIMITED EXTENT MANAGEMENT LOCAL;

-- 创建审计表空间
CREATE TABLESPACE AUDIT_DATA DATAFILE '/u01/app/oracle/oradata/HRDB/audit_data01.dbf' SIZE 500M AUTOEXTEND ON NEXT 100M MAXSIZE UNLIMITED LOGGING EXTENT MANAGEMENT LOCAL SEGMENT SPACE MANAGEMENT AUTO;
```

2. **创建数据库用户**：

```sql
-- 创建系统管理员用户
CREATE USER SYSTEM_ADMIN IDENTIFIED BY "ComplexPassword123!" DEFAULT TABLESPACE SYSTEM_DATA TEMPORARY TABLESPACE TEMP_DATA;
GRANT CONNECT, RESOURCE, DBA TO SYSTEM_ADMIN;

-- 创建应用用户
CREATE USER HR_APP IDENTIFIED BY "AppPassword456!" DEFAULT TABLESPACE USER_DATA TEMPORARY TABLESPACE TEMP_DATA;
GRANT CONNECT, RESOURCE TO HR_APP;
GRANT CREATE SEQUENCE, CREATE VIEW, CREATE PROCEDURE, CREATE TRIGGER TO HR_APP;
GRANT SELECT ON V_$SESSION TO HR_APP;
GRANT SELECT ON V_$SQLAREA TO HR_APP;

-- 创建只读用户
CREATE USER HR_READ ONLY IDENTIFIED BY "ReadPassword789!" DEFAULT TABLESPACE USER_DATA TEMPORARY TABLESPACE TEMP_DATA;
GRANT CONNECT TO HR_READ_ONLY;
GRANT SELECT ON HR_APP.EMPLOYEE TO HR_READ_ONLY;
GRANT SELECT ON HR_APP.ORGANIZATION TO HR_READ_ONLY;
-- 根据需要授予其他表的SELECT权限
```

3. **配置数据库参数**：

```sql
-- 设置SGA和PGA大小
ALTER SYSTEM SET sga_target=16G SCOPE=SPFILE;
ALTER SYSTEM SET pga_aggregate_target=4G SCOPE=SPFILE;

-- 设置进程数和会话数
ALTER SYSTEM SET processes=300 SCOPE=SPFILE;
ALTER SYSTEM SET sessions=335 SCOPE=SPFILE;

-- 设置打开的游标数
ALTER SYSTEM SET open_cursors=300 SCOPE=SPFILE;

-- 设置日志缓冲区大小
ALTER SYSTEM SET log_buffer=16M SCOPE=SPFILE;

-- 启用归档日志模式
SHUTDOWN IMMEDIATE;
STARTUP MOUNT;
ALTER DATABASE ARCHIVELOG;
ALTER DATABASE OPEN;

-- 配置归档日志位置
ALTER SYSTEM SET log_archive_dest_1='LOCATION=/u01/app/oracle/archivelog' SCOPE=BOTH;
```

### 3.2 应用服务器安装

#### 3.2.1 WebLogic服务器安装

1. **安装前准备**：
   - 以root用户登录Linux服务器
   - 创建weblogic用户和组
   - 安装Java JDK（推荐JDK 11）
   - 设置环境变量

2. **安装WebLogic软件**：
   - 以weblogic用户登录服务器
   - 解压WebLogic安装包
   - 创建响应文件silent.xml，包含安装配置
   - 运行静默安装命令：`java -jar fmw_14.1.1.0.0_wls.jar -silent -responseFile ./silent.xml -invPtrLoc ./oraInst.loc`
   - 等待安装完成

3. **创建WebLogic域**：
   - 切换到WebLogic安装目录下的wlserver/common/bin
   - 创建域配置响应文件domain.properties
   - 运行域创建命令：`./config.sh -silent -responseFile ./domain.properties -invPtrLoc ./oraInst.loc`
   - 等待域创建完成

#### 3.2.2 应用部署

1. **准备部署包**：
   - 获取人事子系统WAR包（HRSystem.war）
   - 验证WAR包完整性

2. **通过WebLogic控制台部署**：
   - 启动WebLogic管理服务器
   - 访问WebLogic控制台（http://服务器IP:7001/console）
   - 使用管理员账号登录
   - 导航至"部署"页面
   - 点击"安装"按钮
   - 选择WAR包文件路径
   - 点击"下一步"，选择"安装为应用程序"
   - 选择部署目标服务器
   - 配置应用名称为"HRSystem"
   - 配置上下文路径为"/hr"
   - 完成部署配置并启动应用

3. **通过命令行部署**：
   - 切换到WebLogic域目录下的bin
   - 执行部署命令：`./wlst.sh deploy.py`
   - deploy.py脚本内容示例：

```python
connect('weblogic', 'password', 't3://localhost:7001')
startEdit()
appPath = '/path/to/HRSystem.war'
部署应用程序
appName = 'HRSystem'
deploy(appName, appPath, targets='AdminServer', stageMode='stage', planPath=None)
activate()
disconnect()
```

### 3.3 安全配置

#### 3.3.1 服务器安全加固

1. **操作系统安全加固**：
   - 安装最新安全补丁
   - 禁用不必要的服务和端口
   - 配置防火墙规则
   - 设置SSH密钥认证，禁用密码登录
   - 配置文件系统权限
   - 启用日志审计

2. **数据库安全加固**：
   - 修改默认账户密码
   - 移除不必要的数据库账户
   - 限制数据库账户权限
   - 启用数据库审计
   - 配置数据库加密（TDE）
   - 设置网络加密（SSL/TLS）

3. **应用服务器安全加固**：
   - 修改默认管理账户密码
   - 移除示例应用
   - 配置SSL/TLS
   - 限制管理控制台访问
   - 启用WebLogic审计
   - 配置内存保护（JVM安全参数）

#### 3.3.2 SSL/TLS配置

1. **生成SSL证书**：

```bash
# 生成密钥库
keytool -genkey -alias hrserver -keyalg RSA -keysize 2048 -keystore hrserver.jks -validity 3650

# 导出证书
keytool -export -alias hrserver -keystore hrserver.jks -file hrserver.cer

# 如果使用CA签名，需向CA申请签名
# 导入CA根证书
keytool -import -trustcacerts -alias root -file cacert.cer -keystore hrserver.jks
```

2. **配置WebLogic SSL**：
   - 登录WebLogic管理控制台
   - 导航至"环境" > "服务器" > "AdminServer" > "配置" > "SSL"
   - 设置密钥库路径和密码
   - 设置私钥别名和密码
   - 配置SSL监听端口（默认7002）
   - 保存配置并重启服务器

3. **配置强制HTTPS**：
   - 在WebLogic控制台创建安全约束
   - 配置HTTP重定向到HTTPS
   - 或在应用程序web.xml中配置安全约束

## 4️⃣ 系统初始化配置

### 4.1 基本配置

#### 4.1.1 系统参数配置

1. **访问管理控制台**：
   - 打开浏览器，访问系统管理控制台：https://服务器IP:7002/hr/admin
   - 使用默认管理员账号登录（用户名：admin，密码：Admin123!）

2. **配置系统基本信息**：
   - 导航至"系统设置" > "基本信息"
   - 设置系统名称："GMP环境下人事管理子系统"
   - 设置系统描述："符合GMP要求的企业人事信息管理系统"
   - 设置系统版本："v1.0.0"
   - 设置公司名称："[公司名称]"
   - 上传公司Logo
   - 保存配置

3. **配置系统参数**：
   - 导航至"系统设置" > "参数配置"
   - 设置会话超时时间：30分钟
   - 设置密码过期时间：90天
   - 设置登录失败锁定次数：5次
   - 设置数据备份频率：每天
   - 设置审计日志保留期限：1年
   - 设置最大上传文件大小：10MB
   - 保存配置

#### 4.1.2 安全参数配置

1. **密码策略配置**：
   - 导航至"系统设置" > "安全设置" > "密码策略"
   - 设置密码最小长度：8位
   - 设置密码复杂度要求：必须包含大小写字母、数字和特殊字符
   - 设置密码历史记录：不允许使用最近5次的密码
   - 启用密码定期更改提醒
   - 保存配置

2. **认证方式配置**：
   - 导航至"系统设置" > "安全设置" > "认证方式"
   - 启用用户名密码认证
   - 可选：配置LDAP/AD认证
   - 可选：配置双因素认证
   - 保存配置

3. **会话安全配置**：
   - 导航至"系统设置" > "安全设置" > "会话安全"
   - 启用会话绑定IP
   - 启用并发会话限制：每个用户最多2个并发会话
   - 设置会话空闲超时：15分钟
   - 保存配置

```sql
-- 验证员工表记录数
SELECT '源系统员工数' AS 数据源, COUNT(*) AS 记录数 FROM OLD_EMPLOYEE_TABLE
UNION ALL
SELECT '目标系统员工数' AS 数据源, COUNT(*) AS 记录数 FROM HR_APP.EMPLOYEE;

-- 验证各部门员工数
SELECT 
    d.部门名称,
    (SELECT COUNT(*) FROM OLD_EMPLOYEE_TABLE e WHERE e.DEPT_CODE = d.DEPT_CODE) AS 源系统员工数,
    (SELECT COUNT(*) FROM HR_APP.EMPLOYEE e WHERE e.department_id = (SELECT id FROM HR_APP.DEPARTMENT WHERE code = d.DEPT_CODE)) AS 目标系统员工数
FROM
    OLD_DEPARTMENT_TABLE d
ORDER BY
    d.部门名称;
```

### 8.3 问题管理

#### 8.3.1 问题分类与优先级

1. **问题分类**：
   - 系统错误：影响系统正常运行的错误
   - 功能问题：功能缺失或不符合需求
   - 性能问题：系统响应慢或资源消耗过高
   - 数据问题：数据不准确或不完整
   - 安全问题：潜在的安全漏洞或风险

2. **优先级划分**：
   - P0（紧急）：系统完全不可用，影响所有用户
   - P1（高）：关键功能不可用，影响大量用户
   - P2（中）：重要功能受影响，影响部分用户
   - P3（低）：非关键功能问题，影响少数用户
   - P4（建议）：优化建议或小改进

#### 8.3.2 问题处理流程

1. **问题接收**：
   - 接收用户报告的问题
   - 记录问题详情（现象、影响范围、复现步骤）
   - 初步评估问题严重程度

2. **问题分析**：
   - 分析问题原因
   - 确认问题可复现性
   - 评估解决影响和风险
   - 制定解决方案

3. **问题解决**：
   - 实施解决方案
   - 验证解决效果
   - 记录解决方案和步骤

4. **问题关闭**：
   - 确认问题解决
   - 获取用户反馈
   - 更新问题状态
   - 归档问题记录

#### 8.3.3 常见问题及解决方案

| 问题现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 系统登录失败 | 用户名或密码错误 | 重置用户密码 |
| | 账号被锁定 | 解锁用户账号 |
| | 网络连接问题 | 检查网络连接和服务器状态 |
| 页面加载缓慢 | 服务器负载过高 | 优化服务器配置或增加资源 |
| | 数据库查询效率低 | 优化SQL查询和索引 |
| | 前端资源过大 | 压缩前端资源，启用缓存 |
| 数据保存失败 | 数据库连接问题 | 检查数据库连接和服务状态 |
| | 数据验证失败 | 检查输入数据是否符合要求 |
| | 权限不足 | 检查用户权限设置 |
| 报表生成失败 | 报表配置错误 | 检查和修正报表配置 |
| | 数据量过大 | 优化报表查询，增加查询条件 |
| | 服务器内存不足 | 增加服务器内存配置 |
| 文件上传失败 | 文件格式不支持 | 检查文件格式是否符合要求 |
| | 文件大小超限 | 调整上传文件大小限制 |
| | 存储空间不足 | 清理磁盘空间或增加存储容量 |
