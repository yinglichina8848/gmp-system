3// ============================================================================
4// GMP系统用户组织角色关联实体定义
5// dịp
6//
7// WHY: 传统的RBAC模型不支持用户在多个组织内拥有不同角色的复杂权限场景，
8// 在GMP企业中，一个质量工程师可能同时在质量组织、生产组织、培训组织中
9// 担任不同角色，这种多组织多角色的权限需求需要专门的实体进行管理。
10//
11// WHAT: UserOrganizationRole实体实现用户、组织、角色三者间的关联管理，
12// 支持临时权限、权限过期、分配审核等高级权限管理特性，确保权限
13// 分配的追溯性和合规性，满足GMP审计要求。
14//
15// HOW: 使用唯一约束保证用户在同一组织内只能有一个角色，通过枚举约束赋值状态，
16// 使用审计字段记录完整的权限变更历史，支持权限的动态激活和自动失效。
17// ============================================================================
18

188 @Transient
189 public boolean isEffective() {

191 if (!status.isActiveStatus()) {
192 return false;
193 }
194
195 // 2. 检查生效时间

200
201 // 3. 检查到期时间
