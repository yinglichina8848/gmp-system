## 1. 概述

本文档描述了生产执行系统(MES)的用例模型和用户故事，旨在通过不同角度描述系统功能需求，确保开发团队和业务团队对系统功能有一致的理解。文档包括用例图、用例描述和用户故事三个主要部分。

## 2. 术语定义

| 术语 | 解释 |
| ---- | ---- |
| MES | 制造执行系统(Manufacturing Execution System)，用于监控和管理生产执行过程的信息系统 |
| GMP | 药品生产质量管理规范(Good Manufacturing Practice)，确保药品质量的一系列规范和标准 |
| 批次 | 一次生产活动中生产的具有相同特性和质量的产品集合，通常有唯一的批次编号 |
| 工单 | 生产任务的基本单位，包含生产什么、生产多少、何时完成等信息 |
| 工艺路线 | 产品从原材料到成品的加工步骤和方法的描述 |
| BOM | 物料清单(Bill of Materials)，描述产品由哪些原材料组成及用量 |

## 3. 用户角色定义

| 角色名称 | 职责描述 |
| ---- | ---- |
| 系统管理员 | 负责系统配置、用户管理、权限分配和系统维护 |
| 生产主管 | 负责生产计划制定、生产订单管理和生产进度监控 |
| 生产计划员 | 负责具体生产计划的排产和调整，生成生产工单 |
| 操作员 | 执行具体生产操作，记录生产数据和执行结果 |
| 质量主管 | 负责质量管理体系的运行和监督，审核质量记录 |
| 质量检验员 | 执行产品和物料的检验，记录检验结果 |
| 设备管理员 | 负责设备维护计划制定和设备状态监控 |
| 仓库管理员 | 负责原材料入库、成品出库和库存管理 |
| 生产经理 | 负责生产部门的整体管理，审批重要生产决策 |
| 工艺工程师 | 负责工艺流程设计和工艺参数优化 |
| 数据分析师 | 负责生产数据的分析和报表生成 |
| IT支持人员 | 负责系统技术支持和问题解决 |
| 合规官 | 负责确保系统符合GMP等相关法规要求 |
| 采购经理 | 负责原材料采购和供应商管理 |
| 财务人员 | 负责生产成本核算和财务报表生成 |
| 审计人员 | 负责系统和生产过程的审计工作 |

## 5. 用例描述

### 5.1 生产订单管理用例

#### 5.1.1 创建生产订单

| 用例编号 | UC-ORDER-001 |
| ---- | ---- |
| 用例名称 | 创建生产订单 |
| 参与者 | 生产主管 |
| 前置条件 | 1. 生产主管已登录系统<br>2. 产品主数据已维护<br>3. 原材料库存信息可用 |
| 后置条件 | 1. 生产订单成功创建<br>2. 订单状态为"待审批"<br>3. 系统记录创建操作日志 |
| 主流程 | 1. 生产主管访问生产订单管理页面<br>2. 点击"创建订单"按钮<br>3. 选择产品，输入订单数量、计划开始时间等信息<br>4. 系统自动计算所需原材料数量并检查库存<br>5. 生产主管确认订单信息<br>6. 点击"提交"按钮<br>7. 系统创建订单并提示成功 |
| 备选流程 | 1. 原材料库存不足：系统提示库存不足，用户可选择调整订单数量或取消操作<br>2. 产品信息不存在：系统提示产品不存在，用户需先维护产品信息 |
| 异常流程 | 1. 数据验证失败：系统提示具体错误信息，用户需修正后重新提交<br>2. 系统错误：提示系统错误信息，记录日志，用户可联系IT支持 |
| 业务规则 | 1. 订单数量必须为正数<br>2. 计划时间必须合理（不早于当前时间）<br>3. 原材料库存必须满足生产需求 |
| 输入数据 | 产品ID、订单数量、计划开始时间、计划完成时间、优先级等 |
| 输出数据 | 创建的生产订单信息、订单编号 |
| 扩展点 | 支持从销售订单导入信息、支持批量创建订单 |

| 用例编号 | UC-ORDER-002 |

| 用例名称 | 审批生产订单 |
| 参与者 | 生产经理 |

| 后置条件 | 1. 订单状态更新为"已批准"或"已拒绝"<br>2. 系统记录审批操作和意见<br>3. 若批准，系统自动触发后续流程 |
| 主流程 | 1. 生产经理访问审批页面<br>2. 查看待审批订单列表<br>3. 选择一个订单，查看详细信息<br>4. 根据生产计划和资源情况，决定批准或拒绝<br>5. 输入审批意见<br>6. 点击"批准"或"拒绝"按钮<br>7. 系统更新订单状态并提示操作成功 |
| 备选流程 | 1. 要求修改：生产经理可要求修改订单部分信息后重新提交 |
| 异常流程 | 1. 订单已被其他用户处理：系统提示订单状态已变更，需刷新页面<br>2. 系统错误：提示系统错误信息，记录日志，用户可联系IT支持 |
| 业务规则 | 1. 只有生产经理角色可审批订单<br>2. 已拒绝的订单可重新提交审批<br>3. 审批必须在规定时间内完成 |
| 输入数据 | 审批决定（批准/拒绝）、审批意见 |
| 输出数据 | 更新后的订单状态、审批记录 |
| 扩展点 | 支持多级审批流程、支持审批通知 |

### 5.2 批次执行管理用例

#### 5.2.1 创建生产批次

| 用例编号 | UC-BATCH-001 |
| ---- | ---- |
| 用例名称 | 创建生产批次 |
| 参与者 | 生产计划员 |
| 前置条件 | 1. 生产计划员已登录系统<br>2. 存在已批准的生产订单<br>3. 生产资源已分配 |
| 后置条件 | 1. 生产批次成功创建<br>2. 生成唯一的批次号<br>3. 批次状态为"准备中"<br>4. 系统自动生成生产指令 |
| 主流程 | 1. 生产计划员访问批次管理页面<br>2. 选择已批准的生产订单<br>3. 点击"创建批次"按钮<br>4. 系统自动生成批次号，填充基本信息<br>5. 生产计划员分配生产线、设备和人员<br>6. 设置批次优先级和计划开始时间<br>7. 点击"确认创建"按钮<br>8. 系统创建批次并提示成功 |
| 备选流程 | 1. 资源冲突：系统提示资源已被占用，用户可调整分配或时间<br>2. 订单数量拆分：可根据需要将一个订单拆分为多个批次 |
| 异常流程 | 1. 数据验证失败：系统提示具体错误信息，用户需修正后重新提交<br>2. 系统错误：提示系统错误信息，记录日志，用户可联系IT支持 |
| 业务规则 | 1. 批次号必须唯一<br>2. 资源分配不能冲突<br>3. 批次数量不能超过订单剩余数量 |
| 输入数据 | 订单ID、生产线、设备、人员、优先级、计划时间等 |
| 输出数据 | 创建的批次信息、批次号、生产指令 |
| 扩展点 | 支持基于规则自动分配资源、支持批量创建批次 |

#### 5.2.2 批次生产执行

| 用例编号 | UC-BATCH-002 |
| ---- | ---- |
| 用例名称 | 批次生产执行 |
| 参与者 | 操作员 |
| 前置条件 | 1. 操作员已登录系统<br>2. 批次状态为"准备中"<br>3. 生产指令已生成<br>4. 物料和设备已准备就绪 |
| 后置条件 | 1. 批次状态更新为"执行中"<br>2. 生产数据实时记录到系统<br>3. 工序状态更新<br>4. 关键操作有电子签名 |
| 主流程 | 1. 操作员登录系统，访问批次执行页面<br>2. 扫描批次条形码或选择待执行批次<br>3. 系统显示批次生产指令和工艺参数要求<br>4. 操作员开始执行生产操作<br>5. 按要求记录工艺参数和生产数据<br>6. 关键操作点进行电子签名确认<br>7. 完成当前工序，系统自动流转到下一工序<br>8. 所有工序完成后，批次状态更新为"待检验" |
| 备选流程 | 1. 参数异常：系统提示参数超出范围，操作员需确认或调整<br>2. 生产中断：操作员可暂停批次执行，记录中断原因 |
| 异常流程 | 1. 数据录入错误：系统提示数据验证失败，需重新输入<br>2. 设备异常：系统触发设备异常处理流程<br>3. 电子签名失败：系统提示签名失败原因，操作员需重新签名 |
| 业务规则 | 1. 数据录入必须实时准确<br>2. 关键操作必须有电子签名<br>3. 工艺参数必须在允许范围内<br>4. 工序必须按顺序执行 |
| 输入数据 | 工艺参数值、操作时间、执行结果、电子签名等 |
| 输出数据 | 更新的批次状态、生产数据记录、操作日志 |
| 扩展点 | 支持数据自动采集、支持异常情况处理流程 |

#### 5.2.3 批次完成确认

| 用例编号 | UC-BATCH-003 |
| ---- | ---- |
| 用例名称 | 批次完成确认 |
| 参与者 | 生产主管 |
| 前置条件 | 1. 生产主管已登录系统<br>2. 批次所有工序已完成<br>3. 质量检验合格<br>4. 所有生产数据已完整记录 |
| 后置条件 | 1. 批次状态更新为"已完成"<br>2. 系统生成批次记录报告<br>3. 成品自动入库<br>4. 相关生产订单状态更新 |
| 主流程 | 1. 生产主管访问批次管理页面<br>2. 查看待完成确认的批次列表<br>3. 选择一个批次，查看所有工序完成情况和质量检验结果<br>4. 确认所有生产数据完整无误<br>5. 点击"确认完成"按钮并进行电子签名<br>6. 系统更新批次状态为"已完成"<br>7. 系统自动生成批次记录报告<br>8. 成品数量自动转入库存系统 |
| 备选流程 | 1. 数据不完整：系统提示缺失数据，生产主管需补充完整后重新提交<br>2. 质量不合格：系统提示质量问题，批次进入不合格品处理流程 |
| 异常流程 | 1. 签名失败：系统提示签名失败原因，需重新签名<br>2. 系统错误：提示系统错误信息，记录日志，用户可联系IT支持 |
| 业务规则 | 1. 只有生产主管可确认批次完成<br>2. 所有数据必须完整记录<br>3. 质量检验必须合格<br>4. 完成确认后数据不可修改 |
| 输入数据 | 电子签名、完成确认 |
| 输出数据 | 更新的批次状态、批次记录报告、库存更新 |
| 扩展点 | 支持批量确认完成、支持批次总结报表 |

### 5.3 设备监控用例

#### 5.3.1 设备状态监控

| 备选流程 | 1. 设备离线：系统提示设备连接状态异常，用户可联系维护人员<br>2. 数据异常：系统自动标记异常数据，用户可查看详细信息 |

| 业务规则 | 1. 设备状态数据应实时更新<br>2. 异常情况必须及时告警<br>3. 历史数据应保留一定时间周期 |
| 输入数据 | 监控参数、告警阈值设置 |
| 输出数据 | 设备状态列表、详细状态信息、历史数据趋势、告警信息 |
| 扩展点 | 支持设备效率分析、支持预测性维护 |

### 5.4 物料追溯用例

#### 5.4.1 物料接收与检验

| 用例编号 | UC-MATERIAL-001 |
| ---- | ---- |
| 用例名称 | 物料接收与检验 |
| 参与者 | 仓库管理员、质量检验员 |
| 前置条件 | 1. 用户已登录系统<br>2. 物料已送达<br>3. 供应商信息完整 |
| 后置条件 | 1. 物料信息录入系统<br>2. 生成物料批次号<br>3. 检验结果记录并更新物料状态<br>4. 合格物料入库 |
| 主流程 | 1. 仓库管理员接收物料，确认供应商和数量<br>2. 登录系统，创建物料接收记录<br>3. 系统生成物料批次号<br>4. 仓库管理员将物料移至待检区<br>5. 质量检验员接收检验任务，执行检验<br>6. 录入检验数据和结果<br>7. 系统根据检验结果更新物料状态<br>8. 合格物料移至合格品库，不合格物料进行隔离处理 |
| 备选流程 | 1. 部分合格：可对物料进行部分合格判定<br>2. 让步接收：经审批后的不合格物料可让步接收 |
| 异常流程 | 1. 检验数据异常：系统提示异常，检验员需重新检验<br>2. 系统错误：提示系统错误信息，记录日志，用户可联系IT支持 |
| 业务规则 | 1. 物料必须经检验合格后方可入库使用<br>2. 检验数据必须真实准确<br>3. 不合格物料必须进行隔离处理 |
| 输入数据 | 物料信息、供应商信息、数量、检验数据、检验结果 |
| 输出数据 | 物料接收记录、物料批次号、检验报告、物料状态更新 |
| 扩展点 | 支持扫码接收、支持检验标准自动匹配 |

## 6. 用户故事

### 6.1 生产订单管理用户故事

#### 6.1.1 创建生产订单

**作为** 生产主管，
**我希望** 能够快速创建生产订单，
**以便** 及时安排生产任务，响应市场需求。

**验收标准：**
- 能够从产品目录中选择产品，自动带出产品信息
- 能够设置订单数量、优先级和计划时间
- 系统能够自动检查原材料库存是否充足
- 创建的订单自动进入待审批状态
- 支持批量导入订单信息

#### 6.1.2 审批生产订单

**作为** 生产经理，
**我希望** 能够审批生产订单，
**以便** 确保生产计划符合公司资源和战略目标。

**验收标准：**
- 能够查看待审批订单列表及详细信息
- 能够根据实际情况批准或拒绝订单
- 能够添加审批意见和说明
- 审批后订单状态正确更新
- 支持多级审批流程

#### 6.1.3 跟踪生产订单进度

**作为** 生产主管，
**我希望** 能够实时跟踪生产订单的执行进度，
**以便** 及时调整生产计划，确保按时交付。

**验收标准：**
- 能够按订单号、产品、时间段等条件查询订单
- 能够直观查看订单的完成百分比
- 能够查看每个工序的执行状态和负责人
- 能够查看订单的预计完成时间
- 提供订单执行报表导出功能

### 6.2 批次执行管理用户故事

#### 6.2.1 创建生产批次

**作为** 生产计划员，
**我希望** 能够基于生产订单创建生产批次，
**以便** 组织具体的生产活动。

**验收标准：**
- 能够从已批准的订单中选择创建批次
- 系统能够自动生成唯一的批次号
- 能够分配生产线、设备和操作人员
- 能够设置批次优先级和计划时间
- 能够根据需要将一个订单拆分为多个批次

#### 6.2.2 批次生产执行

**作为** 操作员，
**我希望** 能够按照系统提示执行生产操作并记录数据，
**以便** 确保生产过程符合标准，数据完整准确。

**验收标准：**
- 能够扫描批次号快速找到待执行任务
- 系统显示详细的操作指导和工艺参数要求
- 能够方便地记录实际工艺参数和操作结果
- 关键操作点需要电子签名确认
- 异常情况能够及时记录和上报

#### 6.2.3 批次完成确认

**作为** 生产主管，
**我希望** 能够确认批次生产完成，
**以便** 完成生产闭环，生成批次记录。

**验收标准：**
- 能够查看批次所有工序的完成情况
- 能够确认所有生产数据完整无误
- 完成确认后自动生成批次记录报告
- 成品数量自动转入库存系统
- 相关生产订单状态自动更新

### 6.3 设备监控用户故事

#### 6.3.1 设备状态监控

**作为** 设备管理员，
**我希望** 能够实时监控设备运行状态和参数，
**以便** 及时发现和处理设备异常，确保生产正常进行。

**验收标准：**
- 能够查看所有设备的实时状态（运行、停机、维护等）
- 能够查看设备的关键运行参数和趋势图
- 能够设置参数阈值和告警条件
- 异常情况能够自动触发告警通知
- 能够查看设备历史运行数据

#### 6.3.2 设备维护管理

**作为** 设备管理员，
**我希望** 能够管理设备维护计划和执行过程，
**以便** 延长设备使用寿命，减少故障率。

**验收标准：**
- 能够制定设备定期维护计划
- 能够创建和分配维护任务
- 维护人员能够记录维护内容和结果
- 能够查看设备的完整维护历史
- 维护完成后设备状态自动更新

### 6.4 物料追溯用户故事

#### 6.4.1 物料接收与检验

**作为** 仓库管理员，
**我希望** 能够记录物料接收信息并触发检验流程，
**以便** 确保入库物料符合质量要求。

**验收标准：**
- 能够快速记录物料接收信息（供应商、数量、日期等）
- 系统能够自动生成物料批次号
- 能够将物料移至待检区并通知检验员
- 能够查看检验结果和物料状态
- 合格物料能够自动入库

#### 6.4.2 物料使用追踪

**作为** 操作员，
**我希望** 能够记录物料的使用情况，
**以便** 跟踪物料流向，确保物料使用的准确性。

**验收标准：**
- 能够扫描物料条码快速记录使用情况
- 能够记录物料的实际使用数量和用途
- 系统能够自动扣减物料库存
- 能够建立物料与生产批次的关联关系
- 超耗情况能够记录并触发审批

#### 6.4.3 产品追溯查询

**作为** 质量主管，
**我希望** 能够查询产品的完整生产历史和物料来源，
**以便** 在质量问题发生时能够快速定位原因，采取措施。

**验收标准：**
- 能够通过产品批次号查询完整的生产过程
- 能够查看使用的原材料批次信息
- 能够查看生产设备、操作人员和工艺参数
- 能够查看质量检验记录和结果
- 支持正向和反向追溯功能
- 能够导出完整的追溯报告

## 7. 总结

本文档通过用例模型和用户故事两个维度详细描述了生产执行系统(MES)的功能需求。用例模型从系统功能的角度描述了系统与用户之间的交互，而用户故事则从用户需求的角度描述了系统应该提供的价值。这两种描述方式相互补充，共同确保开发团队和业务团队对系统功能有全面一致的理解，为后续的系统设计、开发和测试提供了基础。
