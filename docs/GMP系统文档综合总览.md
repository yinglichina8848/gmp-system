# GMP系统文档综合总览

## 1. 系统概述

GMP信息管理系统是一套完整的符合药品GMP（Good Manufacturing Practice）规范的企业管理平台，旨在帮助制药企业实现生产过程的规范化、数据化和可追溯性，确保产品质量符合法规要求。系统采用微服务架构设计，包含多个功能相互协作的子系统，共同构成完整的GMP合规管理体系。

## 3. 系统架构总览

### 3.1 微服务架构

GMP系统采用现代微服务架构设计，各子系统相对独立，通过RESTful API进行通信。系统主要包含以下层次：

- **表现层**：Web前端应用、移动应用
- **API网关**：路由、负载均衡、认证授权
- **业务逻辑层**：各功能子系统
- **数据访问层**：数据持久化和访问
- **数据存储层**：关系型数据库、NoSQL数据库、对象存储
- **基础设施层**：容器化、编排、监控、日志

### 3.2 技术栈

- **后端框架**：Spring Boot、Spring Cloud
- **数据库**：PostgreSQL、MongoDB、Redis
- **消息队列**：RabbitMQ
- **缓存**：Redis
- **存储**：MinIO
- **搜索引擎**：Elasticsearch
- **工作流引擎**：Camunda BPM
- **前端框架**：React、Vue.js、Angular
- **容器化**：Docker、Kubernetes
- **监控**：Prometheus、Grafana
