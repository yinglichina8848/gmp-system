# EDMS子系统文档总览

## 1. 子系统概述
EDMS（Electronic Document Management System）电子文档管理系统是GMP系统中的文档管理核心，负责电子文档的全生命周期管理，确保文档的合规性、版本控制和可追溯性。作为工程文档管理系统的专业基础平台，提供文档存储、审批、工作流等通用能力。

## 3. 核心功能模块

- **文档管理**：文档创建、分类、存储、版本控制全生命周期管理
- **工作流引擎**：灵活的文档审批流程和电子签名
- **权限控制**：基于角色的细粒度访问控制和操作权限管理
- **版本控制**：多版本文档管理、差异比较和历史追溯
- **全文检索**：强大的文档内容搜索和元数据过滤
- **合规审计**：符合GMP要求的完整审计追踪和电子签名
- **集成接口**：与各子系统的标准API接口和数据交换

## 4. 技术架构

EDMS采用微服务架构设计，核心技术栈：

- **后端技术**：Java Spring Boot + Spring Cloud微服务框架
- **数据库技术**：PostgreSQL（元数据）+ Elasticsearch（全文检索）
- **存储系统**：分布式对象存储（兼容多个云存储方案）
- **前端技术**：React + Ant Design组件库
- **工作流引擎**：Camunda BPM（支持复杂审批流程）
- **认证安全**：集成AUTH-SYS的JWT认证和权限控制

## 5. 关键业务流程

- **文档审批流程**：文档标准化模板 → 起草审核 → 多级审批 → 电子签名 → 发布生效
- **版本管控流程**：版本创建 → 变更审核 → 通知分发 → 历史保留
- **权限访问流程**：角色验证 → 权限检查 → 操作记录 → 审计追踪

## 6. 系统集成关系

作为GMP系统的基础平台，EDMS与各子系统深度集成：
- **为QMS提供**：SOP管理、质量手册、变更控制文档存储
- **为MES提供**：工艺规程、技术说明等生产指导文档
- **为LIMS提供**：分析方法、标准操作规程等实验室文档
- **为设备管理提供**：设备手册、维护规程、校验记录
- **为人事系统提供**：培训资料、资格证书等文档管理

## 7. 设计理念

### 7.1 GMP合规优先
- 构建符合《中国药典》GMP附录计算机化系统的电子文档管理
- 完整的审计追踪：谁在何时对什么文档做了什么操作
- 电子签名和数字证书支持，确保操作的可追溯性

### 7.2 用户体验为中心
- 直观的文档导航和强大的搜索能力
- 移动端兼容，支持现场查阅关键文档
- 智能推荐：基于用户角色和工作内容的文档推荐

### 7.3 高度可扩展
- 支持企业私有化部署和云上SaaS部署
- 丰富的API接口，便于与其他业务系统的深度集成
- 支持多语言和国际化扩展
