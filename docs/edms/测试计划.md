## 概述

### 文档目的
本文档定义了GMP系统文档管理子系统(EDMS)的测试计划，包括测试范围、策略、环境、用例和进度安排等内容，确保EDMS系统符合GMP法规要求并满足业务需求。

### 参考文档
- [EDMS系统总体需求](总体需求.md)
- [EDMS系统架构设计](架构设计.md)
- [EDMS系统详细需求](详细需求.md)
- [EDMS系统用户故事](用例与用户故事.md)

### GMP特殊要求
- **合规性验证**: 所有功能必须符合GMP法规要求
- **审计可追溯**: 关键操作必须支持完整审计追踪
- **数据完整性**: 文档必须准确存储，防止篡改和丢失
- **权限分离**: 文档操作与审批必须实施职责分离原则

## 测试范围

### 包含的功能模块
1. **文档库管理模块**
   - 文档分类与目录管理
   - 文档存储与版本控制
   - 文档检索与浏览
   
2. **文档生命周期管理模块**
   - 文档起草与审批
   - 文档发布与分发
   - 文档修订与废止
   
3. **文档权限管理模块**
   - 用户/角色权限设置
   - 文档访问控制
   - 操作权限分级管理
   
4. **文档审计追踪模块**
   - 文档操作日志记录
   - 文档变更历史追踪
   - 审计报告生成
   
5. **文档检索与分析模块**
   - 全文检索功能
   - 文档统计分析
   - 报表生成与导出

### 不包含的内容
- OCR文字识别功能
- 复杂的文档内容分析功能
- 与外部文档系统的实时同步

## 测试策略

### 测试类型
1. **单元测试**
   - 验证各服务类的方法逻辑正确性
   - 覆盖率目标：85%以上
   
2. **集成测试**
   - 验证各模块间的接口调用正确性
   - 验证数据库操作正确性
   
3. **接口测试**
   - 验证RESTful API的功能正确性
   - 验证请求/响应数据格式
   
4. **业务流程测试**
   - 验证文档从起草到废止的完整生命周期
   - 验证文档审批流程
   - 验证文档版本控制功能
   
5. **安全测试**
   - 验证文档访问权限控制
   - 验证文档数据完整性保护
   
6. **性能测试**
   - 验证大文档上传下载性能
   - 验证并发文档检索性能

### 测试方法
- **黑盒测试**: 验证功能是否符合需求
- **白盒测试**: 验证代码逻辑是否正确
- **灰盒测试**: 结合前后端验证业务流程

## 测试用例

### 功能测试用例

#### TC001: 文档上传与版本控制
- **前置条件**: 用户具有文档上传权限
- **测试步骤**:
  1. 登录系统，进入文档管理模块
  2. 选择目标文档库，点击"上传文档"
  3. 选择本地文件上传
  4. 填写文档基本信息
  5. 提交文档
  6. 再次上传同名文档（新版本）
- **预期结果**: 
  1. 文档成功上传并存储
  2. 系统自动生成文档编号
  3. 文档版本正确标识（V1.0, V2.0）
  4. 旧版本文档仍可访问
  5. 操作日志完整记录

#### TC002: 文档审批流程
- **前置条件**: 存在待审批的文档
- **测试步骤**:
  1. 审批人员登录系统
  2. 查看待审批文档列表
  3. 选择文档进行审批
  4. 填写审批意见并提交
- **预期结果**: 
  1. 审批流程正确触发
  2. 审批人员能正确接收到审批任务
  3. 审批后文档状态正确更新
  4. 审批记录完整保存

#### TC003: 文档检索功能
- **前置条件**: 系统中存在多种类型文档
- **测试步骤**:
  1. 登录系统，进入文档检索页面
  2. 输入关键词进行全文检索
  3. 使用筛选条件缩小检索范围
  4. 查看检索结果
- **预期结果**: 
  1. 检索结果准确且响应迅速
  2. 支持按文档类型、时间等条件筛选
  3. 支持相关性排序
  4. 能检索到文档内容中的关键词

### 安全测试用例

#### TC004: 文档访问权限控制
- **前置条件**: 系统中存在不同密级的文档
- **测试步骤**:
  1. 使用普通用户账号尝试访问机密文档
  2. 使用无权限账号尝试下载受控文档
- **预期结果**: 
  1. 系统拒绝未授权访问
  2. 提示相应权限不足信息

#### TC005: 文档完整性保护
- **前置条件**: 系统中存在重要文档
- **测试步骤**:
  1. 尝试直接修改数据库中文档存储路径
  2. 尝试删除文件系统中的文档文件
  3. 尝试访问被删除的文档
- **预期结果**: 
  1. 系统具备文档完整性保护机制
  2. 文档不会因非法操作而丢失或损坏
  3. 异常操作被记录并告警
