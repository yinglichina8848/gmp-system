# 电子文档管理系统（EDMS）

## 1. 项目简介

电子文档管理系统（EDMS - Electronic Document Management System）是GMP医药企业信息管理系统的核心组成部分，负责电子文档的全生命周期管理。该系统确保文档的合规性、版本控制和可追溯性，涵盖文档创建、审批、发布、分发、使用、归档和销毁等完整流程。

## 2. 系统功能概述

- **文档管理**：提供完整的文档创建、编辑、存储和管理功能
- **文件存储管理**：集成的文件存储服务，支持大文件上传、下载和管理
- **版本控制**：支持文档版本历史记录、比较和回滚操作
- **工作流与审批**：内置灵活的工作流引擎，支持复杂的审批流程
- **权限与电子签名**：实现细粒度的访问控制和符合法规的电子签名
- **发布与归档**：支持文档的正式发布和长期归档管理
- **全文检索**：基于Elasticsearch的高性能文档内容搜索
- **审计追踪**：完整的操作日志和变更历史记录
- **对象存储**：基于MinIO的分布式对象存储，支持高并发和高可用
- **系统集成**：提供标准化的RESTful API接口和向后兼容的文件服务API

## 3. 技术架构

EDMS系统采用现代化的微服务架构设计，基于以下核心技术栈：

- **后端**：Java 17, Spring Boot 3.x, Spring Cloud 2022.x
- **数据库**：PostgreSQL 14
- **缓存**：Redis 7.x
- **搜索引擎**：Elasticsearch 8.x
- **对象存储**：MinIO (集成文件存储服务)
- **消息队列**：RabbitMQ
- **前端**：Vue.js 3.x, Element Plus
- **容器化**：Docker, Kubernetes

## 5. 快速开始

### 5.1 环境准备
- JDK 17+
- Maven 3.8+
- Docker & Docker Compose
- Node.js 16+ (前端开发)

### 5.2 启动服务
```bash
# 启动依赖服务
docker-compose -f infrastructure/docker-compose.yml up -d

# 编译和启动后端服务
cd services/edms-service
mvn spring-boot:run

# 编译和启动前端服务
cd frontend
npm install
npm run dev
```

### 5.3 访问系统
- **后端API**：http://localhost:8085
- **前端界面**：http://localhost:3000
- **API文档**：http://localhost:8085/swagger-ui.html
