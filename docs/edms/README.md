# EDMS子系统文档总览

## 1. 子系统概述
EDMS（Electronic Document Management System）电子文档管理系统是GMP系统中的文档管理核心，负责电子文档的全生命周期管理，包括文档创建、审批、发布、分发、使用、归档和销毁，确保文档的合规性、版本控制和可追溯性。

## 3. 核心功能模块

- **文档管理**：文档创建、分类、存储、检索和版本控制
- **工作流管理**：文档审批流程定义、执行和监控
- **权限管理**：细粒度的文档访问权限控制和操作权限管理
- **版本控制**：文档版本历史记录、差异比较和版本回溯
- **电子签名**：符合21 CFR Part 11的电子签名和审计追踪
- **文档发布**：正式文档的发布、通知和分发管理
- **文档归档**：文档生命周期结束后的归档管理和长期保存
- **文档检索**：高级搜索功能，支持全文检索和元数据过滤

## 4. 技术架构

EDMS子系统采用微服务架构设计，主要技术栈包括：

- **后端技术**：Java、Spring Boot、Spring Cloud
- **数据库**：PostgreSQL（元数据）、MongoDB（非结构化数据）
- **存储技术**：MinIO（对象存储）、分布式文件系统
- **前端技术**：React、Ant Design
- **API设计**：RESTful API
- **认证授权**：集成AUTH-SYS子系统的JWT认证机制
- **搜索引擎**：Elasticsearch（全文检索）
- **工作流引擎**：Camunda BPM（文档审批流程）

## 5. 关键业务流程

- **文档生命周期流程**：文档创建 → 审核 → 批准 → 发布 → 使用 → 变更 → 归档
- **文档审批流程**：提交审批 → 审核人审核 → 审批决策 → 通知 → 状态更新
- **文档变更流程**：变更申请 → 影响评估 → 变更实施 → 审核批准 → 重新发布

## 6. 系统接口

EDMS子系统与其他子系统的主要接口包括：

- **与AUTH-SYS集成**：用户认证、授权和角色管理
- **与QMS集成**：质量文档（SOP、质量手册等）管理
- **与MES集成**：生产相关文档的关联和访问
- **与LIMS集成**：实验方法和标准操作规程管理
- **与EQUIPMENT-MANAGEMENT集成**：设备文档和维护手册管理
- **与FILE-SERVICE集成**：大文件存储和传输
