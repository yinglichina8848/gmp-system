## 1. 文档目的

本文档旨在为EDMS（Electronic Document Management System，电子文档管理系统）子系统提供全面的文档说明和使用指南。通过本文档，读者可以快速了解EDMS子系统的整体结构、核心功能、技术架构以及相关文档的组织方式，为系统的开发、测试、部署和维护提供指导。

## 2. EDMS子系统概述

EDMS子系统是GMP（Good Manufacturing Practice）信息管理系统中的核心模块，负责电子文档的全生命周期管理，确保文档的合规性、版本控制和可追溯性。作为工程文档管理系统的专业基础平台，EDMS提供文档存储、审批、工作流等通用能力，为GMP系统中的其他业务模块提供文档支持。

### 2.1 核心功能

- **文档管理**：提供文档的创建、分类、存储、检索和归档等全生命周期管理功能
- **版本控制**：实现严格的文档版本管理，支持版本比较、历史追溯和版本回滚
- **工作流审批**：提供灵活的文档审批流程设计和执行功能，支持多级审批和条件分支
- **电子签名**：实现符合GMP和21 CFR Part 11要求的电子签名功能，确保操作的不可否认性
- **权限管理**：基于角色的细粒度权限控制，确保文档的访问安全性
- **全文检索**：提供强大的文档内容和元数据检索功能，支持复杂的检索条件组合
- **审计追踪**：完整记录文档的所有操作历史，支持合规性审计

### 2.2 技术架构

EDMS采用微服务架构设计，主要技术栈包括：

- **后端技术**：Java Spring Boot + Spring Cloud微服务框架
- **数据库技术**：PostgreSQL（元数据）+ Elasticsearch（全文检索）
- **存储系统**：分布式对象存储（兼容多个云存储方案）
- **前端技术**：React + Ant Design组件库
- **工作流引擎**：Camunda BPM（支持复杂审批流程）
- **认证安全**：集成AUTH-SYS的JWT认证和权限控制

文档版本与系统版本保持一致，遵循以下版本号规则：

- 主版本号.次版本号.修订号（例如：1.0.0）
- 主版本号：系统架构或核心功能发生重大变化
- 次版本号：新增功能或改进
- 修订号：修复缺陷或小的改进
