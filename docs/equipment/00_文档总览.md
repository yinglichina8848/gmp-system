# Equipment-Management子系统文档总览

## 1. 子系统概述
Equipment-Management设备管理系统是GMP系统中的设备运维核心组件，负责管理企业所有生产和检测设备的全生命周期，包括设备台账、校准管理、维护计划、故障处理和设备效率分析等，确保设备的合规性、可靠性和可用性。

## 3. 核心功能模块

- **设备台账管理**：设备基本信息、技术参数、位置信息和供应商管理
- **校准管理**：校准计划制定、校准执行、偏差处理和校准证书管理
- **维护管理**：预防性维护计划、维护工单、维护记录和效果评估
- **故障管理**：故障报修、响应、处理和根因分析
- **备件管理**：备件库存、消耗跟踪和采购计划
- **设备效率分析**：OEE计算、设备利用率统计和性能分析
- **设备验证**：设备IQ/OQ/PQ验证管理和文档记录
- **合规性管理**：设备相关法规符合性追踪和报告

## 4. 技术架构

Equipment-Management子系统采用微服务架构设计，主要技术栈包括：

- **后端技术**：Java、Spring Boot、Spring Cloud
- **数据库**：PostgreSQL（主数据库）、Redis（缓存）
- **前端技术**：Vue.js、Element UI
- **API设计**：RESTful API
- **认证授权**：集成AUTH-SYS子系统的JWT认证机制
- **工作流引擎**：Camunda BPM（维护工单和校准流程）
- **数据分析**：集成数据仓库进行设备效率分析
- **移动支持**：支持移动设备进行现场数据采集和工单处理

## 5. 关键业务流程

- **设备校准流程**：校准计划制定 → 校准工单生成 → 校准执行 → 数据记录 → 偏差处理 → 证书生成
- **设备维护流程**：维护计划制定 → 维护工单生成 → 维护执行 → 记录更新 → 效果评估
- **故障处理流程**：故障报修 → 评估分类 → 派工处理 → 维修执行 → 验证 → 关闭

## 6. 系统接口

Equipment-Management子系统与其他子系统的主要接口包括：

- **与AUTH-SYS集成**：用户认证、授权和角色管理
- **与MES集成**：生产设备信息共享和状态监控
- **与LIMS集成**：检测设备信息和校准状态同步
- **与QMS集成**：设备相关质量事件和变更控制
- **与EDMS集成**：设备文档、SOP和维护手册管理
- **与WAREHOUSE-MANAGEMENT集成**：备件库存管理
