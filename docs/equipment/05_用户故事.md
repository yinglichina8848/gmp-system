## 概述

### 文档目的
本文档基于用户故事的形式，详细描述了设备管理系统中各类用户的需求和期望。通过用户故事，可以更清晰地理解不同角色在系统中的使用场景和目标，为系统设计、开发和测试提供指导。本文档遵循INVEST原则（独立性、可协商性、有价值性、可估算性、可测试性、小规模），确保用户故事的质量和可执行性。

### 术语定义
- **用户故事**: 从用户视角描述系统功能的简短叙述，通常采用"作为[角色]，我希望[功能]，以便[目的]"的格式
- **设备**: 用于药品生产、检验、包装等活动的仪器、设备、设施等
- **GMP**: 药品生产质量管理规范(Good Manufacturing Practice)
- **验收标准**: 用于验证用户故事是否完成的具体、可测试的条件
- **设备生命周期**: 设备从采购、安装、使用、维护到报废的完整过程
- **维护**: 为保持设备正常运行而进行的各类活动
- **校准**: 确定测量设备与可追溯到国际或国家标准的测量标准之间的关系的一系列操作
- **验证**: 证明任何程序、生产过程、设备、物料、活动或系统确实能达到预期结果的有文件证明的一系列活动

## 用户故事详情

### EQ-001: 创建设备基础信息

**用户故事**: 作为设备管理员，我希望能够录入新设备的基础信息，以便系统中能够完整记录所有设备数据

**背景**: 新设备采购到货验收后，需要在系统中建立完整的设备档案，为后续的维护、校准和使用提供基础数据支持。

**详细描述**:
- 能够录入设备的基本信息，包括设备编号、名称、型号规格、制造商、供应商、采购日期等
- 能够设置设备的分类、重要程度、所在部门和位置等属性
- 能够上传设备相关文档，如设备说明书、合格证、技术参数表等
- 能够设置设备的关键联系人
- 能够预览和编辑录入的信息，确认无误后保存

**验收标准**:
1. 能够成功录入所有必填的设备基础信息
2. 支持上传PDF、Word、Excel等格式的设备文档
3. 设备信息保存后能够在系统中正确显示
4. 设备编号自动生成且唯一
5. 必填字段未填写时系统给出提示
6. 文档上传成功后能够在线查看

**相关业务规则**:
- 设备编号必须唯一，不得重复
- 关键设备必须上传完整的技术文档
- 设备信息录入后需要经过审核才能生效

### EQ-006: 提交设备采购申请

**用户故事**: 作为设备管理员，我希望能够提交设备采购申请，以便获取所需的新设备

**背景**: 当生产或检验需要新设备时，设备管理员需要向相关部门申请采购新设备，确保生产活动的正常进行。

**详细描述**:
- 能够创建新的采购申请单
- 能够填写设备的详细需求，包括设备名称、类型、用途、数量、预算等
- 能够添加设备的技术要求和规格参数
- 能够上传参考资料或需求说明书
- 能够指定紧急程度和期望到货时间
- 提交后自动进入审批流程

**验收标准**:
1. 能够创建完整的采购申请单并保存
2. 技术要求部分支持详细描述和附件上传
3. 提交后系统自动通知相关审批人
4. 能够在系统中跟踪采购申请的审批状态
5. 支持采购申请的修改和撤回（在审批完成前）

**相关业务规则**:
- 采购申请必须包含明确的设备需求和预算
- 超过一定金额的采购申请需要多级审批
- 紧急采购需要额外的审批流程

### EQ-011: 创建维护计划

**用户故事**: 作为设备管理员，我希望能够创建设备维护计划，以便定期对设备进行保养

**背景**: 为确保设备的正常运行和延长使用寿命，需要根据设备特性和制造商建议，制定定期维护计划。

**详细描述**:
- 能够为单台设备或设备组创建维护计划
- 能够设置维护周期（如每周、每月、每季度等）
- 能够定义维护任务的具体内容和步骤
- 能够指定维护责任人
- 能够设置维护开始日期和下次维护日期
- 能够根据计划自动生成维护任务

**验收标准**:
1. 能够成功创建维护计划并关联到设备
2. 系统能够根据维护计划自动生成维护任务
3. 维护任务能够正确分配给指定的维护人员
4. 能够在系统中查看所有维护计划和任务
5. 支持维护计划的修改和暂停

**相关业务规则**:
- 关键设备必须有更频繁的维护计划
- 维护计划必须符合设备制造商的建议
- 维护计划变更需要记录变更原因

### EQ-017: 报告设备故障

**用户故事**: 作为设备操作员，我希望能够快速报告设备故障，以便及时获得维修支持

**背景**: 设备在使用过程中可能出现各种故障，操作人员需要能够快速简便地报告故障，以便维护人员能够及时处理，减少对生产的影响。

**详细描述**:
- 能够通过简单的界面快速提交故障报告
- 能够选择故障设备和描述故障现象
- 能够上传故障现场照片或视频
- 能够设置故障紧急程度
- 能够查看故障报告的处理状态
- 能够接收到故障处理的进展通知

**验收标准**:
1. 故障报告界面简洁明了，易于操作
2. 能够成功提交包含必要信息的故障报告
3. 支持上传图片和简短视频
4. 提交后系统自动通知相关维护人员
5. 能够在系统中查看故障处理进度

**相关业务规则**:
- 紧急故障必须在发现后立即报告
- 故障报告必须包含足够的信息以便维护人员判断问题
- 关键设备故障需要同时通知设备管理员

### EQ-022: 设备状态变更

**用户故事**: 作为设备管理员，我希望能够更新设备状态，以便准确反映设备当前情况

**背景**: 设备在生命周期中会经历不同的状态，如待验收、使用中、维护中、停用、报废等，需要及时更新设备状态以确保系统数据的准确性。

**详细描述**:
- 能够查看设备当前状态
- 能够将设备状态变更为使用中、维护中、停用、报废等
- 能够记录状态变更的原因和相关说明
- 某些状态变更需要经过审批流程
- 能够查看设备状态变更历史
- 状态变更后自动更新相关权限和功能

**验收标准**:
1. 能够成功执行设备状态变更操作
2. 状态变更记录包含变更时间、原因、操作人等信息
3. 系统能够根据设备状态控制相关操作权限
4. 关键状态变更能够正确触发审批流程
5. 能够查看设备的完整状态变更历史

**相关业务规则**:
- 设备状态变更必须有合理的原因
- 关键设备的状态变更需要审批
- 设备报废必须经过多级审批

### EQ-027: 用户与权限管理

**用户故事**: 作为系统管理员，我希望能够管理用户和权限，以便控制系统访问安全

**背景**: 为确保系统安全和数据保护，需要严格控制用户对系统功能和数据的访问权限，根据职责分配相应的权限。

**详细描述**:
- 能够创建和管理用户账户
- 能够为用户分配系统角色
- 能够查看用户的权限列表
- 能够启用和禁用用户账户
- 能够重置用户密码
- 能够查看用户登录历史

**验收标准**:
1. 能够成功创建和管理用户账户
2. 用户角色分配后，权限立即生效
3. 禁用的用户账户无法登录系统
4. 能够成功重置用户密码
5. 用户登录历史记录完整

**相关业务规则**:
- 系统管理员账户必须有至少两人拥有
- 用户权限必须遵循最小权限原则
- 定期审查用户权限分配

### 术语表
| 术语 | 解释 |
|------|------|
| 用户故事 | 从用户视角描述系统功能的简短叙述，通常采用"作为[角色]，我希望[功能]，以便[目的]"的格式 |
| 验收标准 | 用于验证用户故事是否完成的具体、可测试的条件 |
| GMP | 药品生产质量管理规范，确保药品质量的一系列标准和规范 |
| 设备生命周期 | 设备从采购、安装、使用、维护到报废的完整过程 |
| 维护 | 为保持设备正常运行而进行的各类活动，包括预防性维护和修复性维护 |
| 校准 | 确定测量设备与可追溯到国际或国家标准的测量标准之间的关系的一系列操作 |
| 验证 | 证明任何程序、生产过程、设备、物料、活动或系统确实能达到预期结果的有文件证明的一系列活动 |
| IQ | 安装确认(Installation Qualification)，确认设备已按照设计要求正确安装 |
| OQ | 运行确认(Operational Qualification)，确认设备在预期的操作范围内能够正常运行 |
